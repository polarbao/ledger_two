# {{PROJECT_NAME}} Dev Workflow

Read first:

1. `AGENTS.md`
2. `.agents/docs/workflow-map.md`
3. `.agents/docs/project-profile.md`
4. `.agents/docs/code-standards.md` for code-facing plans
5. Relevant formal project docs under `{{FORMAL_DOCS_PATH}}`
6. Relevant task cards under `{{CODEX_TASK_PATH}}`
7. Relevant source files

## Workflow

1. Confirm objective, non-objective, and affected boundary.
2. Classify high-risk evidence as A/B/C/D.
3. Give impact, risks, and verification commands.
4. Pause before large rewrites, dependency upgrades, destructive operations, or production-impacting changes.
5. For task-card work, execute only the requested task and do not continue to the next task without explicit user instruction.
6. After confirmation, execute in small verifiable steps.

## Output

- 目标与边界
- 影响面
- 风险
- 验证方式
- 下一步
