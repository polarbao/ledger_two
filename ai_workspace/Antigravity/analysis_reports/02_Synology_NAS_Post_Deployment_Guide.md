# 群晖 NAS (192.168.0.115) 部署后操作与运维指南

本指南针对群晖 NAS（访问地址为 `http://192.168.0.115:5000`）上的 Docker 部署，详细阐述服务拉起后的**首次配置、业务验证、数据灾备和日常运维**操作。

---

## 1. 第一步：部署成功后的健康检查 (Health Check)

在 Container Manager 中成功拉起项目后，请首先访问健康检查接口，确认后端服务与 SQLite 数据库已物理联通。

* **访问地址**：`http://192.168.0.115:8088/api/healthz`
* **预期响应**：
  ```json
  {
    "success": true,
    "data": {
      "status": "ok",
      "db": "ok",
      "version": "0.2.0"
    }
  }
  ```
> [!CAUTION]
> **常见故障**：如果检测到 `"db": "error"`，代表容器对挂载的物理路径 `/volume1/docker/ledger-two/data` 没有写权限。请在 File Station 中右键该文件夹，在“属性 -> 权限”中为 `System` 或 `Everyone` 开放读写权限。

---

## 2. 第二步：首次使用引导 (Onboarding)

服务首次冷启动时，SQLite 数据库是空的。此时直接访问前端地址会进入引导页面。

1. **访问地址**：`http://192.168.0.115:8088`
2. **初始化账本**：
   * 页面会自动重定向到 `/init`。
   * 输入账本名称（如 `我们的账本`）。
   * 填入两个成员的用户名与显示名称（如 `user_a` 和 `user_b`）并设定初始密码。
3. **完成引导**：点击确认。系统将在后台自动生成初始的消费分类（如餐饮、交通、娱乐等）和默认账户，并跳转至登录页。

---

## 3. 第三步：核心账务流向与对账验证 (Manual Acceptance)

为确保部署环境稳固、权限完全生效，请依次进行以下核心场景的验证：

### 验证 1：个人隐私账单隔离 (Private Bill Isolation)
1. 用 **成员 A** 登录系统，记录一笔分类为“餐饮”的支出，金额 `100 元`，可见性选择 **仅自己可见 (Private)**。
2. 退出登录，换用 **成员 B** 登录。
3. 检查 B 的流水列表与 Dashboard，确认**看不到** A 记录的这笔 100 元支出。
4. 退出登录。

### 验证 2：共享分摊与自动轧差 (Split & Balance)
1. 用 **成员 A** 登录，记录一笔“家庭超市采购”的支出，金额 `200 元`，分摊方式选择 **平摊 (Equal)**，点击保存。
2. 此时，系统会自动将这笔账单拆分为两份：A 支付了 200 元，A 承担 100 元，B 承担 100 元。
3. 进入**结算中心**页面，系统应当正确提示：“成员 B 欠 成员 A 100.00 元”。

### 验证 3：轧差结清 (Settlement Process)
1. 用 **成员 B** 登录，进入**结算中心**。
2. 点击“**去结算**”或“**发起结算**”。在结算弹窗中确认向 A 支付了 `100.00 元`（例如通过微信/支付宝线下转账后在系统登记）。
3. 确认后，结算状态归零，并在交易流水中生成一条 `settlement` 类型的防篡改对账记录。

---

## 4. 第四步：数据持久性与灾备物理确认 (Data Safety)

### 1. 验证重启不丢数据
1. 在大屏随便记录几笔账。
2. 登录群晖 `192.168.0.115:5000`，打开 **Container Manager**。
3. 在“项目”或“容器”中找到 `ledger-two`，点击 **重启 (Restart)**。
4. 容器重启完毕后，重新刷新 `http://192.168.0.115:8088`，确认先前的记账数据和登录状态依然存在，证明挂载卷已接通。

### 2. 验证手动安全备份
1. 登录系统，进入**系统设置 (Settings)** 页面。
2. 找到数据安全区域，点击 **立即创建安全备份**（此时会有高风险二次确认弹窗，点击确认）。
3. 登录群晖 DSM，使用 **File Station** 打开 `/volume1/docker/ledger-two/backups/manual/` 文件夹。
4. 确认是否生成了以当前时间戳命名的 `.db` 镜像备份文件（如 `backup_20260611_xxxxxx.db`）。

---

## 5. 第五步：日常运维与排错 (Maintenance)

### 1. 实时日志查看
如果在运行中遇到报错，可以通过群晖控制台或 SSH 终端查看实时运行日志：
```bash
# SSH 登入 NAS 后运行
docker logs -f ledger-two
```
或者在挂载的物理路径 `/volume1/docker/ledger-two/logs/` 目录下，直接使用文本编辑器打开系统的日志文件。

### 2. 以后如何进行版本升级？
当发布了新版本（如 v0.3 或更高版本）时，群晖上的无损升级步骤如下：
1. 备份数据库：将 `/volume1/docker/ledger-two/data/ledger.db` 下载到本地保存。
2. 登录 SSH 终端，进入项目目录，执行：
   ```bash
   cd /volume1/docker/ledger-two
   docker-compose down
   docker-compose pull  # 拉取最新镜像
   docker-compose up -d --build
   ```
3. 系统冷启动后，Go 服务会自动检测并运行数据库新字段的迁移（goose up），无缝升级数据结构。
