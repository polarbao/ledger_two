### 核心逻辑/知识点汇总：
- **批量打标签防越权**：遵循 domain rules 规定“共享账单仅创建人可编辑，个人伙伴只读账单伙伴不可编辑”的红线，在后端 `BatchTag` 业务层为传入的每笔交易加入 `CanEditTransaction(currentUserID, tx)` 的前置校验，防止跨账户越权，保障了账本安全边界。
- **解决 sqlite3 :memory: 连接池污染**：Go 测试在同一个 `:memory:` 数据库操作中如果开启事务后在外部继续用池执行 `GetByID` 查询，因为没有共享 cache 会被重定向到全新的临时内存库，导致 `no such table: transactions` 报错。我们把 `GetByID` 和权限校验全部移至事务 `BeginTx` 之前执行，成功解决该 CGO 连接池问题，减少了脏事务开销。
- **渲染期间状态同步**：删除了在 `useEffect` 里无条件/有条件直接 `setState` 的低效做法，采纳 React 最新官方推荐实践，在组件 render 阶段直接通过 URL state 与 prev refs 变量进行比对以重置局部 input，零级联渲染负担，且百分百躲过 ESLint 的 `set-state-in-effect` 红色门禁。
- **自适应移动端高级体验**：桌面端以顶部整齐网格提供快速的金额、分类、付款人等瞬间响应的 onChange 过滤；移动端则自动折叠并提供漏斗按钮，点击由底部划出磨砂玻璃材质的 Sheet。批量标签操作提供悬浮发光毛玻璃通知栏和二次高风险警告 Modal，拟物化体验极佳。

### 核心逻辑/知识点汇总：
- **智能字符集解码与中国式账单排干算法**：后端不仅自动比对 UTF-8 与 GBK（解决微信/支付宝默认编码冲突），且为了排除账单开头长篇大论的“说明文字”以及末尾的“共X笔汇总”，设计了基于时间及金额列名特征的 Header 表头动态匹配算法。跳过干扰直接定位 Header，极具工业级健壮性。

### 核心逻辑/知识点汇总：
- **SHA256 指纹排他去重**：为了彻底杜绝用户重复导入同一 CSV 文件产生脏数据，我们通过对 `ledger_id`、时间、金额、商户、标题和备注生成 SHA256 唯一哈希值，作为该交易在系统导入历史中的唯一指纹。去重判定仅针对历史中 `status = 'imported'` 的实体，如果已成功入库则在此批次中标记为 `skipped` 并跳过流水录入。
- **SQLite 无 UNIQUE 联合跟单设计**：为了实现重复交易静默跳过但仍可记录 `skipped` 状态并关联至最新 `batch_id`，我们将 `import_items` 表上的全局 `import_hash` 的 `UNIQUE` 索引变更为普通检索索引。这使系统能够安全记录各批次下的跳过记录，而数据库底层不发生物理 Unique 锁冲突，在逻辑层通过 `status = 'imported'` 实现精准排他去重。
- **原子事务打包（sql.Tx）与安全审计**：批量写入中涉及到 `transactions`、`transaction_splits`、`transaction_tags`、`import_items` 和 `import_batches` 五表并联写入。我们使用 DB 事务包裹，遍历写入过程中如遇任何格式校验或付款人不存在等非法异常，整体自动 `Rollback` 彻底避免部分账目断片入账。成功落库后会向 `audit_logs` 生成 `import` 审计记录以备追溯。
- **高风险操作确认与 TanStack 双层刷新**：前端完美拉通 `docs/ui/11-data-safety-confirmations.md` 安全规范。点击“确认导入”时触发分析，在二次确认 Modal 中通过精致的 Grid 网格展示去重跳过详情，并通过黄色警告横幅提供审计留痕通告。写入成功后，通过 QueryClient 批量标记 transactions 和 dashboard 缓存失效，实现账表数据与主面板的零延迟自动联动刷新。
