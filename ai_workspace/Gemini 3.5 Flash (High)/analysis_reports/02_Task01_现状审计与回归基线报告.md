# Task 01: Demo 现状审计与回归基线报告

## 1. 总体审计结论
当前项目已完成 v0.1 Demo 阶段，已完全跑通核心闭环：包括系统的环境引导（初始化）、两用户登录鉴权、流水/共同账单增删改查、结算及债务差额计算、仪表盘统计数据展示等。项目整体的逻辑分层明确，设计思路具有较强的前瞻性，达到了可用于本地联调与后续稳定性迭代的标准。

---

## 2. 核心模块现状审计

### 2.1 后端模块 (Backend)
* **实现现状**：
  - 基于 `Go + chi v5` 构建轻量级 HTTP 服务。
  - 数据模型层 (Repository)、业务逻辑层 (Service) 与传输控制器层 (Handler) 进行了清晰的解耦，杜绝了底层与协议层循环依赖。
  - 用户登录鉴权采用 JWT Token，存放在安全属性强的 `HttpOnly Cookie` 中，有效防止客户端脚本攻击 (XSS)。
  - 已支持普通账单的 CRUD、共同支出的记账（包含 split 分摊逻辑）与结算 settlements 的录入。
  - 看板 API 已集成对于成员间净债务 (Net balance) 的综合计算。
* **主要隐患与待办**：
  - 静态文件 fallback 处理器 `spaHandler` 锁定了磁盘路径 `./web/dist`，而在本地编译输出为 `frontend/dist`，在本地直接运行后端主程序时可能会遇到前端资源无法读取的问题。该设计目前只在 Docker 构建阶段天然对齐。
  - 暂无审计日志拦截器或写入逻辑，需要等待后续 v0.2 的任务落地。

### 2.2 前端模块 (Frontend)
* **实现现状**：
  - 技术栈为 `React + TypeScript + Vite`。
  - 路由挂载了全局守卫 `AppInitGuard`，确保未初始化时跳转 `/init`、未登录时跳转 `/login`，已登录后挂载主框架布局 `AppShell`。
  - 页面结构完整，包含：`InitPage` (初始化配置)、`LoginPage` (登录)、`DashboardPage` (账目看板)、`SettlementPage` (结算流水录入与结清)、`TransactionsPage` (明细列表)、`AnalyticsPage` 与 `SettingsPage` 等。
  - 基于 TanStack Query 管理服务端请求状态，基于 Zustand 处理局部交互状态，数据流向清晰。
* **主要隐患与待办**：
  - 账目明细页目前只包含基础骨架，前端与后端交互的很多复杂查询（如模糊检索、时间段过滤等）在 Demo 阶段尚未完全联调完毕。

### 2.3 数据库层 (Database)
* **实现现状**：
  - 使用 SQLite 本地单文件数据库（默认存储于 `backend/data/ledger.db`）。
  - 集成了 `goose` 作为数据库版本控制工具，迁移配置 `001_init.sql` 中已经建立包括 `users`、`ledgers`、`accounts`、`categories`、`tags`、`transactions`、`transaction_splits`、`settlements` 和 `audit_logs` 的完整 Schema。
  - 表结构极具前瞻性，已经在表定义中提前注入了 `ledger_id`，为未来 v0.5 多账本迁移打下了优异的架构基础。
* **主要隐患与待办**：
  - 默认情况下本地开发的 `db.go` 没有集成迁移脚本的自动检查与自动升级，主要由独立的 `cmd/migrate` 命令行工具跑迁移。

### 2.4 部署与容器化 (Deploy)
* **实现现状**：
  - 根目录下配置了 `docker-compose.yml`，对 SQLite 数据文件 (data/)、备份 (backups/)、上传资料 (uploads/) 进行了挂载绑定，保证了容器生命周期外的数据安全性。
  - `deploy/docker/Dockerfile` 采用高效的三阶段构建法，极大地压缩了镜像包体积并实现了前后端同容器运行。
* **主要隐患与待办**：
  - 本地开发环境下测试时，由于 PATH 缺失或平台架构差异，若不经过特定的环境变量配置，可能会导致 `pnpm build` 时无法寻址 Node 引擎。

## 3. 可运行命令汇总 (Mac/Unix 环境)

为了回归项目基线并校验工程健康度，我们汇总并校验了以下在本地可以直接运行的命令：

### 3.1 数据库迁移
```bash
# 切换至 backend 目录
cd backend
# 校验迁移状态
/opt/homebrew/bin/go run cmd/migrate/main.go status
# 执行数据库表升级
/opt/homebrew/bin/go run cmd/migrate/main.go up
```

### 3.2 后端单元测试
```bash
# 执行全部后端集成与单元测试
/opt/homebrew/bin/go test ./...
```

### 3.3 前端构建与代码校验
```bash
# 切换至 frontend 目录
cd frontend
# 导入临时环境变量以保证 node 和 pnpm 运行
export PATH="/opt/homebrew/bin:$PATH"
# 安装依赖
pnpm install
# 执行编译打包
pnpm build
# 运行 lint 检测
pnpm lint
```
