# 02 UI 交互设计稿：LedgerTwo v0.2

## 1. 设计目标

基于 Figma/原型中的清新绿色风格，升级为适合长期使用的双人共享账本 Web 应用。

设计重点：

1. 不只展示账单，而是突出“付款、承担、分摊、结算”。
2. 桌面端信息密度更高，移动端保留卡片轻量体验。
3. 流水、统计、结算分离，避免首页过长。
4. 新增账单需要支持快捷录入和高级分摊。
5. NAS 本地服务状态需要可感知。

移动端保留你截图中的卡片风格，但增加底部导航。

```text
┌──────────────────────┐
│ 共享账本        已同步 │
│ 2025年4月   总支出    │
├──────────────────────┤
```

## 9. 页面五：统计分析 Analytics

统计页包含四个 Tab。

```text
统计分析
├── 趋势
├── 分类
├── 成员
└── 标签
```

### 9.1 趋势统计

展示每月总支出变化。

```text
2025年1月 ¥2,100
2025年2月 ¥2,680
2025年3月 ¥3,120
2025年4月 ¥3,280
```

### 9.2 分类统计

| 分类 | 金额 | 占比 |
|---|---:|---:|
| 餐饮 | ¥1,470 | 44.8% |
| 购物 | ¥850 | 25.9% |
| 交通 | ¥490 | 14.9% |
| 娱乐 | ¥320 | 9.7% |

点击分类后跳转流水页并自动带上分类筛选。

### 9.3 成员统计

展示两个人的：

- 实际付款金额
- 实际承担金额
- 净垫付金额
- 个人消费金额
- 共同消费金额

### 9.4 标签统计

展示午餐、外卖、超市、电影、通勤、咖啡、旅行等标签的金额排行。

标签不作为一级分类，而作为横向辅助维度。

## 11. 状态设计

### 11.1 空状态

```text
本月还没有账单
和 lynn 记下第一笔共同支出吧
[记一笔]
```

### 11.2 加载状态

使用骨架屏：

- 金额卡片灰色占位
- 流水列表灰色占位
- 图表区域灰色占位

### 11.3 删除确认

```text
删除后会影响本月结算金额，是否继续？
[取消] [确认删除]
```

### 11.4 服务异常

```text
无法连接到账本服务
请检查 NAS 服务是否运行
```

### 11.5 同步状态

| 状态 | 文案 |
|---|---|
| 正常 | 本地服务正常，最后同步：2025-04-12 21:30 |
| 同步中 | 正在保存到账本服务 |
| 失败 | 保存失败，请重试 |
| 离线 | 无法连接 NAS 服务 |
