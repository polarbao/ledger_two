# 测试与验收规格 v0.3

## 1. 测试目标

账本项目的核心风险是：

1. 金额计算错误。
2. 分摊计算错误。
3. 权限泄露。
4. 结算逻辑破坏历史数据。
5. 数据库迁移不可重复。

因此 Demo 版本必须优先测试业务逻辑，而不是只测试 UI。

## 2. 后端单元测试

### 2.1 金额测试

| 用例 | 输入 | 期望 |
|---|---|---|
| 金额为 0 | 0 | 创建失败 |
| 金额为负 | -100 | 创建失败 |
| 金额为 1 分 | 1 | 创建成功 |
| 金额为 10001 分平均分摊 | payer=A | A=5001, B=5000 |

### 2.2 共同支出测试

#### 用例 A：A 支付 200，两人平摊

```text
A paid = 20000
A share = 10000
B share = 10000
结果：B 欠 A 10000
```

#### 用例 B：B 支付 80，两人平摊

```text
B paid = 8000
A share = 4000
B share = 4000
结果：A 欠 B 4000
```

#### 合并结果

```text
B 欠 A 6000
```

### 2.3 结算测试

在上述基础上，B 向 A 结算 6000：

```text
settlement: B -> A 6000
结果：双方结清
```

### 2.4 权限测试

| 场景 | 期望 |
|---|---|
| A 创建 private 账单 | B 不可见 |
| A 创建 partner_readable 账单 | B 可见，不可编辑 |
| A 创建 shared 账单 | B 可见 |
| B 修改 A 创建的 shared 账单 | Demo 禁止 |

### 2.5 删除测试

1. 删除普通账单后列表不可见。
2. 删除 shared_expense 后结算余额重新计算。
3. 删除操作写入 audit_logs。
4. deleted 数据不参与统计。

## 3. API 集成测试

建议使用 Go `httptest`。

### 3.1 流程测试

```text
POST /api/init/setup
POST /api/auth/login
POST /api/transactions
POST /api/shared-expenses
GET  /api/settlements/balance
POST /api/settlements
GET  /api/settlements/balance
```

### 3.2 期望

1. 所有响应结构符合 `{success,data}` 或 `{success,error}`。
2. 未登录访问返回 401。
3. 参数错误返回 400。
4. 无权限返回 403。

## 5. 手工验收清单

### 5.1 初始化

- [ ] 第一次访问进入初始化页。
- [ ] 可以创建两个用户。
- [ ] 初始化后不能再次访问 setup。

### 5.2 登录

- [ ] A 可以登录。
- [ ] B 可以登录。
- [ ] 错误密码提示失败。
- [ ] 退出后无法访问 API。

### 5.3 流水

- [ ] 可以新增支出。
- [ ] 可以新增收入。
- [ ] 可以编辑自己创建的账单。
- [ ] 可以软删除账单。
- [ ] 列表按月份筛选正确。

### 5.4 共同支出

- [ ] 可以选择付款人。
- [ ] 可以选择平均分摊。
- [ ] 可以选择仅付款人承担。
- [ ] 分摊结果展示正确。

### 5.5 结算

- [ ] A 支付 200，两人平摊，B 欠 A 100。
- [ ] B 支付 80，两人平摊，A 欠 B 40。
- [ ] 合并后 B 欠 A 60。
- [ ] B 结算 60 后双方结清。

### 5.6 统计

- [ ] 本月总支出正确。
- [ ] 分类统计正确。
- [ ] 标签统计正确。
- [ ] 成员支付/承担金额正确。

### 5.7 部署

- [ ] 本地 `docker compose up -d --build` 成功。
- [ ] 容器重启后数据不丢。
- [ ] NAS 上能访问。
- [ ] 可以导出 CSV。
- [ ] 可以触发备份。

## 6. 最小 CI 建议

GitHub Actions 后续可加入：

```text
backend:
  go test ./...
frontend:
  pnpm install
  pnpm lint
  pnpm test
  pnpm build
```

Demo 早期可以先手动执行。
