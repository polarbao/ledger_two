## RC02 部署环境与数据库隔离

状态：本地实现完成；NAS production 一致性备份完成，原库仍为 schema 12

目标：

1. 增加 `DEPLOYMENT_CHANNEL` 并在 health、系统诊断和前端显示。
2. Compose 强制显式容器名、镜像标签和部署 channel。
3. NAS staging 与 production 使用不同目录、端口、容器、密钥和数据库。
4. 提供 SQLite 一致性预部署备份脚本与校验清单。

完成标准：

1. 非法 channel 拒绝启动。
2. staging 显示“验收环境”，production 显示“正式数据”。
3. staging migration 只修改数据库副本，production schema 12 原库保持不变。
4. 备份副本通过 `PRAGMA quick_check`，并记录 schema 与核心表数量。
