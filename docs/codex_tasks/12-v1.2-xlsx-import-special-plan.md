## 1. 专项目标

在 v1.2 正式发布前，为微信和支付宝增加 `.xlsx` 原始账单文件支持。所有 XLSX 流水必须继续经过预览、规则建议、去重、人工调整和单事务提交。

专项不改变：

1. owner-only 权限。
2. 金额整数分。
3. duplicate/suspicious/invalid 语义。
4. transaction_import_refs 唯一去重。
5. commit 单事务和审计日志。
6. staging 先验收、production 后提交的真实数据边界。

## 9. Task49X.6：XLSX 运行开关与降级门禁

状态：已完成代码、全量测试、本机 Docker 重部署和 health 回读（本轮提交）。

目标：让 XLSX reader 可以独立关闭，不影响 CSV、历史预览批次和 commit 主链路。

实现规则：

1. `IMPORT_XLSX_ENABLED` 只接受 `true/false`，非法值拒绝启动。
2. 未显式配置时，development/staging 默认开启，production 默认关闭。
3. production 必须在 schema 19 staging 验收后显式设为 `true`。
4. 开关关闭时，微信/支付宝 XLSX 返回 `IMPORT_FILE_UNSUPPORTED`，CSV 继续可用且不创建失败批次。
5. `/api/healthz` 返回 `import_xlsx_enabled`，作为部署验收证据。
