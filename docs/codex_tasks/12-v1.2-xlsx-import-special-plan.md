# Task49X：v1.2 XLSX 导入专项任务计划

状态：开发前准备完成，待进入 Task49X.1
目标版本：v1.2
建立日期：2026-07-12
前置：Task47-Task49 已完成，schema 18，现有 CSV 主链路保持冻结

## 1. 专项目标

在 v1.2 正式发布前，为微信和支付宝增加 `.xlsx` 原始账单文件支持。所有 XLSX 流水必须继续经过预览、规则建议、去重、人工调整和单事务提交。

专项不改变：

1. owner-only 权限。
2. 金额整数分。
3. duplicate/suspicious/invalid 语义。
4. transaction_import_refs 唯一去重。
5. commit 单事务和审计日志。
6. staging 先验收、production 后提交的真实数据边界。

## 2. 提交策略

每个下列 Task 完成后独立提交；Task 内部只有在代码与测试必须分离时再拆提交。提交信息继续使用仓库当前“emoji + type(scope) + 完整条目 + 验证”风格。

建议提交边界：

1. 文档准备独立提交。
2. Task49X.1、49X.2、49X.3、49X.4、49XU 各独立提交。
3. staging 真实文件验收只提交脱敏报告，不提交原始文件。

## 3. Task49X.1：格式边界与测试基座

目标：先建立失败测试、匿名 fixture 和依赖边界，不写数据库业务。

开发范围：

1. 引入并固定 `github.com/xuri/excelize/v2`。
2. 新增 CSV/XLSX 格式枚举和扩展名 + magic + OOXML 结构检测。
3. 新增匿名微信、支付宝 XLSX fixture 和异常结构 fixture。
4. 新增格式不匹配、generic+xlsx、加密/宏/异常结构测试。
5. 新增 XLSX 运行开关，默认 production 关闭。

完成标准：

- [ ] `go.mod/go.sum` 固定评审版本，许可证记录已更新。
- [ ] 真实文件未进入 Git。
- [ ] 合法 XLSX、伪装 ZIP、伪装 CSV、`.xls/.xlsm` 均有测试。
- [ ] 所有错误返回稳定错误码。
- [ ] 原有 CSV parser/service 测试继续通过。

验证：

```text
go test ./internal/importer/...
go test ./internal/config/... ./internal/errors/...
```

## 4. Task49X.2：表格读取层重构

目标：把文件容器读取与来源标准化分离，保持 CSV 行为完全兼容。

开发范围：

1. 新增 `TabularDocument/TabularRow/TabularCell`。
2. 将现有 CSV reader 迁入 tabular reader。
3. 抽出 `ParseTabularRows`，微信/支付宝/通用标准化函数继续复用。
4. `PreviewCSV` 迁移为 `PreviewFile`，必要时保留兼容 wrapper。
5. row_number 改为 reader 提供的物理行号，并冻结 CSV 兼容口径。

完成标准：

- [ ] 三个现有 CSV fixture 输出不变。
- [ ] preview、规则、去重、行调整和 commit 测试不回归。
- [ ] handler 只做 HTTP 读取，service 负责格式分发。
- [ ] parser 不访问数据库。

验证：

```text
go test ./internal/importer/...
go test ./internal/...
```

## 5. Task49X.3：微信 XLSX 与 migration 019

目标：完成微信 XLSX 从上传到预览的后端闭环。

开发范围：

1. 使用 Excelize reader 和显式解压限制读取工作簿。
2. 在可见工作表前 50 行定位唯一微信表头。
3. 支持表头第 18 行、物理行号、日期、金额和订单号文本。
4. 拒绝公式、合并数据区、隐藏数据和多候选工作表。
5. 新增 migration 019：file_format、parser_metadata_json。
6. PreviewBatch/OpenAPI 增加格式和解析元数据。
7. 新增微信 XLSX/CSV 标准化及 hash 等价测试。

完成标准：

- [ ] 微信 XLSX fixture 可生成 ready 批次。
- [ ] preview 前后 transactions 数量不变。
- [ ] 表头 18 的首条数据 row_number 为 19。
- [ ] schema 从 18 升到 19，历史 batch 默认 file_format=csv。
- [ ] CSV 与 XLSX 相同流水 hash 一致。

验证：

```text
go test ./internal/importer/... ./internal/database/...
go test ./internal/...
```

## 6. Task49X.4：支付宝 XLSX 适配

目标：在真实匿名样本约束下完成支付宝 XLSX 支持。

开发范围：

1. 根据匿名真实样本冻结表头别名和表头位置。
2. 覆盖交易创建时间/付款时间、类型、金额、交易号和备注。
3. 完成支付宝 XLSX/CSV 标准化与 hash 等价测试。
4. 将未知官方变体返回为可行动错误，不做宽松猜测。

完成标准：

- [ ] 匿名真实样本字段评审完成。
- [ ] 支付宝 fixture 与真实样本均可 preview。
- [ ] 金额、日期和交易号逐项抽样一致。
- [ ] CSV/XLSX 等价测试通过。

阻塞条件：

没有支付宝匿名真实 XLSX 样本时，可以完成 parser 框架和合成 fixture，但不得将支付宝 XLSX 标记为正式冻结。

## 7. Task49XU：前端与 UI/UX

目标：让用户能正确选择、理解和修正 XLSX 导入，而不是只放开文件后缀。

开发范围：

1. 微信/支付宝支持 `.csv,.xlsx`，generic 仅 `.csv`。
2. 上传区与设置入口从“CSV 导入”改为“账单文件导入”。
3. 预览摘要展示格式、工作表、表头行和识别行数。
4. 覆盖格式不匹配、加密、超限、缺表头、多候选和不安全单元格文案。
5. 补充 Figma handoff 中的 XLSX 上传与错误状态。

完成标准：

- [ ] owner 可选择微信/支付宝 XLSX。
- [ ] generic XLSX 在上传前被明确拒绝。
- [ ] 用户能确认识别的是哪张工作表和多少行。
- [ ] 375px/390px 无横向滚动和文本溢出。
- [ ] 前端测试与 build 通过。

验证：

```text
npm test -- --run
npm run build
```

## 8. Task49X.5：staging 真实微信预览验收

目标：使用已只读预检的 276 行微信 XLSX 验证真实数据，但不直接写 production。

顺序：

1. 备份 staging 数据库并记录 health/schema/channel。
2. 上传真实文件到 staging，仅创建 preview。
3. 核对文件 SHA-256、工作表、表头 18、数据行 276。
4. 核对 new/duplicate/suspicious/invalid/skipped 数量。
5. 分层抽样首尾记录、金额、日期、订单号、退款、转账和支付方式。
6. 未经用户确认不调用 commit。
7. 输出脱敏验收报告，不保存文件和完整原文。

完成标准：

- [ ] 276 行数量守恒，差异逐行可解释。
- [ ] 金额总计按收支类型核对一致。
- [ ] transactions 在 preview 前后不变。
- [ ] production 数据库未发生变化。
- [ ] 用户已看到预览摘要并决定是否提交。
