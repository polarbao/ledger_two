## 3. 推荐范围

Task50 建议只正式化“一个账号管理多个独立账本”的可信闭环：

1. 创建、切换、重命名、归档和恢复账本。
2. owner/editor/viewer 角色和成员管理不变量。
3. 所有业务数据、查询缓存、导入批次、导出和备份按账本隔离。
4. 当前账本失效后的明确回退和无账本空状态。
5. 归档账本默认只读；恢复后重新开放写入。

推荐 Task50 不增加多人分摊范围：

1. 不新增 ratio/shares/custom 分摊 UI。
2. 不新增最小转账建议算法。
3. 不实现公开邀请、链接邀请、邮件/短信/微信通知。
4. 不改变 settlement、历史共同账单和导入金额口径。
5. 新增第三名及以上成员是否开放，必须在 PRD 中单独冻结；推荐留到 Task51，避免成员数量先扩张而分摊与可见性语义没有闭环。
