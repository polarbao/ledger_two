## 1. Goal

在不改变 CSV/XLSX 解析、导入去重、整批事务提交和账务金额口径的前提下，完成“默认元数据 -> 分级归类 -> 人工修正 -> 显式学习 -> 后续复用”的确定性闭环。

## 2. Scope

1. migration 022、稳定 `system_key`、规则来源/应用模式和预览解释持久化。
2. `basic_cn_v1` / `empty` 默认元数据模板，覆盖首次初始化、新建账本和既有账本显式应用。
3. 人工、批量、用户规则、学习规则、内置建议和兜底的分级 resolver。
4. 行级解释、重新分类、批量接受/应用、显式记住商户和元数据保护 API。
5. Fresh Light 导入工作台、规则管理、基础包预览和兜底分类替代交互。
6. 匿名 Fixture、迁移守恒、浏览器、feature flag、回滚和发布准入。

## 3. Non-goals

1. 不引入云端 AI/LLM、外部分类 API、本地 ML 或后台隐式学习。
2. 不增加银行同步、OCR、正则/脚本规则、任意 AND/OR 规则编辑器。
3. 不实现二级分类、标签分组、分类合并、跨账本规则复制或自动账户学习。
4. 不修改历史 migration 001-021，不直接升级 WSL/NAS 数据库。
5. Task50 已关闭；Task53 实现期间不推进 Task51/Task52 代码，不改变两人约束。

## 4. Current state

1. importer 已有稳定 preview/commit、selected/suggested 字段和 Task49 规则 CRUD。
2. 现有规则只写建议，首个命中规则生效，缺少来源、置信度、冲突和解释状态。
3. Task53.1 已让初始化和默认新账本原子创建 `basic_cn_v1`，新账本也可显式选择 `empty`。
4. metadata 已有 Owner-only CRUD、排序、归档和恢复，可作为默认包与兜底保护基础。
