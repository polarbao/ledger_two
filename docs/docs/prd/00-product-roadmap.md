# 产品定位与版本路线（Post-Task30 Foundation）

状态：供审核  
适用阶段：Task30 已完成，v1.1 业务需求尚未定稿

## 1. 产品定位

LedgerTwo 是一个面向个人、情侣、夫妻、家庭和小范围固定成员的私有化生活记账系统。产品核心不是企业会计，也不是复杂投资资产管理，而是帮助用户长期记录生活流水、共同支出、分摊关系、结算记录、分类标签和数据备份。

LedgerTwo 的长期产品目标：

1. 个人可以独立记账。
2. 情侣/夫妻可以共同管理生活支出。
3. 家庭成员可以在同一账本内协作记账和分摊。
4. 数据优先保存在用户自己的 NAS / 本地环境中。
5. 数据可备份、可导出、可迁移、可恢复。
6. 记账、分摊、结算和统计由后端统一计算，避免前端口径漂移。

## 2. 当前完成状态

Task01-Task30 已完成，项目可视为 v1.0/MVP 阶段完成。当前已具备或已有雏形的能力包括：

- 登录和初始化。
- 账单流水。
- 共同支出。
- 多人分摊方法。
- 结算中心和结算建议。
- 统计报表。
- CSV/JSON 导入导出。
- SQLite 备份与恢复流程。
- Docker Compose / NAS 部署。
- PWA / 移动端基础体验和离线草稿。
- 多账本和成员角色的初步能力。

但当前文档和基础框架仍需收口：

1. 文档中仍有 Demo/v0.2/v0.3 与 v1.0 混杂的问题。
2. 配置安全、环境变量、Cookie/HTTPS 策略需要统一。
3. 多账本上下文和 RBAC 需要成为统一框架，而不是散落在各模块中。
4. 分类、标签、支付账户管理能力需要补足。
5. API 契约和 OpenAPI 需要补齐。
6. 前端 active ledger、query key 和权限门禁需要规范化。
7. 测试矩阵需要覆盖多账本、权限、迁移、导出、附件和回归。

## 3. 阶段路线

| 阶段 | 目标 | 说明 |
|---|---|---|
| v1.0 / Task30 | MVP 完成 | 已完成，代码已提交至 GitHub |
| Foundation before v1.1 | 基础框架补齐 | 当前阶段，先修正文档事实源和架构基础，不实现未审核 v1.1 业务 |
| v1.1 | 业务扩展 | 等待产品审核后确定，可能包括账号创建模式、邀请机制、共同支付同步、分类标签增强等 |
| v1.2+ | 体验与数据增强 | 预算、提醒、跨端、统计增强、导入智能化等按后续规划推进 |

## 4. Foundation before v1.1 范围

### 4.1 必做

1. 文档事实源收口。
2. 配置与部署安全框架。
3. LedgerContext 与 RBAC 权限框架。
4. API 契约与 OpenAPI 文档框架。
5. 分类、标签、支付账户管理基础能力。
6. 前端账本状态、query key 和 PermissionGate。
7. 设置页信息架构重组。
8. 迁移、测试和质量门禁补齐。
9. 附件访问控制设计。
10. AI/Codex/Gemini 代码风格和任务规范。

### 4.2 暂不做

1. 不实现 v1.1 未审核的完整邀请业务闭环。
2. 不实现公开邀请链接、邮件/短信/微信邀请。
3. 不实现银行自动同步。
4. 不实现企业级组织架构。
5. 不迁移到微服务。
6. 不强制迁移 PostgreSQL。
7. 不做移动原生 App。

## 5. 优先级

| 优先级 | 内容 |
|---|---|
| P0 | 文档事实源、配置安全、LedgerContext/RBAC、API 错误和权限稳定、测试门禁 |
| P1 | 分类/标签/账户管理、前端 query key、设置页重组、OpenAPI、附件访问控制 |
| P2 | 诊断中心、审计日志查询、E2E、开发者工具和文档自动检查 |

## 6. 验收标准

Foundation before v1.1 完成时，必须满足：

1. README、CHANGELOG、PRD、Tech、UI、AI 任务文档对当前阶段表述一致。
2. 后端配置变量和 Docker Compose / `.env.example` 一致。
3. 生产环境无强密钥时服务拒绝启动。
4. 所有业务 API 都能明确解析当前 ledger 上下文。
5. owner/editor/viewer 权限矩阵有文档、有测试、有统一实现入口。
6. 分类、标签、支付账户支持新增、编辑、排序、归档和恢复的基础能力或明确任务计划。
7. 前端切换账本不再依赖强制 reload 的长期方案。
8. API 合同稳定，至少有 OpenAPI 草案或生成计划。
9. CI 覆盖后端测试、前端 lint/test/build、Docker build 和关键业务回归。
10. v1.1 具体业务规划仍保持“待审核”，没有被提前实现。
