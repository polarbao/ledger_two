# PRD：账本、成员、权限模块

## 1. 模块目标

支持 LedgerTwo 从当前固定双人账本，逐步演进为家庭账本、旅行账本、装修账本等多账本场景。

当前阶段 v0.2 仍保持单账本、双成员，v0.5 再引入多账本与多成员。

## 2. 当前 Demo 范围

- 单个默认共享账本。
- 固定两个用户。
- 不开放公开注册。
- 初始化阶段创建两个用户。
- 两个用户均可登录并使用系统。

## 3. 后续功能

### 3.1 多账本

支持创建多个账本：

- 家庭账本。
- 旅行账本。
- 装修账本。
- 育儿账本。
- 临时聚会账本。

每个账本拥有独立成员、分类设置、账单、统计和结算状态。

### 3.2 多成员

从双人扩展为 N 人。多人模式下，需要支持：

- 添加成员。
- 移除成员。
- 成员禁用。
- 成员角色变更。
- 成员结算状态。

### 3.3 角色权限

| 角色 | 权限 |
|---|---|
| owner | 管理账本、成员、分类、导出、删除账本 |
| editor | 记账、编辑自己的账单、参与共同支出 |
| viewer | 只读查看账本和统计 |

## 4. 权限规则

- private 账单仅创建人可见。
- partner_readable 账单对方可见但不可编辑。
- shared 账单双方可见。
- 多账本后，所有 API 必须校验 ledger_id 和 membership。

## 5. 验收标准

- Demo 阶段两个用户可正常登录。
- private 账单不会被对方查询到。
- partner_readable 账单对方只能查看。
- shared 账单双方可查看。
- 后续多账本模式下，A 账本数据不会出现在 B 账本。
