# PRD：共同支出、分摊、结算模块

## 1. 模块目标

解决双人或多人共同消费中的核心问题：谁付款、谁承担、谁需要补给谁、是否已经结清。

## 2. 当前 Demo 范围

- 支持共同支出。
- 支持 equal 平均分摊。
- 支持 payer_only 仅付款人承担。
- 支持双人结算净额计算。
- 支持生成 settlement 结算记录。

## 3. 后续分摊方式

| 分摊方式 | 说明 |
|---|---|
| equal | 平均分摊 |
| payer_only | 仅付款人承担 |
| amount | 按固定金额 |
| ratio | 按比例 |
| shares | 按份数 |
| custom | 自定义分摊 |

## 4. 结算规则

- 每个人净额 = 实际支付金额 - 实际应承担金额 - 已支付结算 + 已收到结算。
- 净额大于 0：该成员垫付。
- 净额小于 0：该成员应补款。
- 结算只生成 settlement 记录，不修改历史共同支出。

## 5. 多人结算方向

v0.5 多成员后，增加最小转账建议。例如 A/B/C 三人中，系统给出最少转账路径，减少手动结算复杂度。

## 6. 验收标准

- A 支付 200，双方平均分摊，B 欠 A 100。
- B 支付 80，双方平均分摊，A 欠 B 40。
- 合并净额为 B 欠 A 60。
- B 向 A 结算 60 后显示双方结清。
- 删除或修改共同支出后，结算状态重新计算。
