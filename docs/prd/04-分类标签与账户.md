# PRD：分类、标签、账户模块

状态：早期模块摘要，保留为历史背景<br>
当前事实源：`docs/prd/34-v1.3-Task53分类标签与导入智能归类.md`、`docs/tech/26-v1.3-Task53分类标签智能化实施契约.md`<br>

> 本文中的默认分类名称、父子分类和自动补全只代表早期方向，不构成 Task53 开工依据。Task53 首期不做二级分类或标签分组，并采用可预览、可选择、幂等的 `basic_cn_v1` / `empty` 默认模板。

## 1. 模块目标

分类、标签和账户用于提升账单组织、筛选和统计能力。

## 2. 分类

默认支出分类：餐饮、购物、交通、娱乐、居家、医疗、旅行、人情、其他。

默认收入分类：工资、奖金、红包、报销、退款、其他。

后续支持：

- 自定义分类。
- 图标与颜色。
- 排序。
- 归档。
- 父子分类，可选。

## 3. 标签

标签用于横向标记，例如午餐、外卖、咖啡、超市、电影、通勤、宝宝用品、房租、水电。

标签支持：

- 创建。
- 自动补全。
- 热门标签展示。
- 点击标签筛选流水。

## 4. 账户

默认账户：现金、微信、支付宝、银行卡、信用卡、其他。

Demo 阶段账户主要用于记录支付来源，不强制做余额校准。

## 5. 验收标准

- 初始化后存在默认分类、标签和账户。
- 新增账单可以选择分类、标签和账户。
- 统计可以按分类和标签聚合。
- 已归档分类不再出现在新增账单选择器中。
