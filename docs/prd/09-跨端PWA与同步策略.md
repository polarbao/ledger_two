# PRD：跨端、PWA、移动端、同步策略模块

## 1. 模块目标

让 LedgerTwo 从响应式 Web 演进为手机可长期使用的记账工具，并为未来 React Native、Tauri 等跨端方案预留接口。

## 2. 跨端路线

| 阶段 | 方案 | 说明 |
|---|---|---|
| 阶段 1 | 响应式 Web | 当前主线 |
| 阶段 2 | PWA | 添加到手机桌面、启动图标、离线壳 |
| 阶段 3 | IndexedDB 缓存 | 缓存最近账单、分类、标签 |
| 阶段 4 | 离线草稿 | 断网保存草稿，联网后同步 |
| 阶段 5 | React Native / Expo | Web 稳定后再考虑 |
| 阶段 6 | Tauri 桌面端 | 可选 |

## 3. 移动端体验

移动端建议采用：

- 首页。
- 流水。
- 记账。
- 结算。
- 我的。

使用底部 Tab 和记账快捷入口。

## 4. 接口原则

- REST JSON。
- 金额单位统一为分。
- 时间统一 ISO8601。
- DTO 不暴露数据库结构。
- 支持分页、筛选、排序。
- 上传接口统一 multipart。
- 认证预留 Cookie + Token 双模式。

## 5. 同步策略

初期：服务端为准，前端在线请求。

后期：IndexedDB 本地缓存、离线草稿、恢复网络后同步、冲突检测。

## 6. 验收标准

- Web 在手机浏览器可正常使用。
- PWA 可添加到手机桌面。
- API 能被 Web、PWA、移动端复用。
- 离线时有明确提示，不丢草稿。
