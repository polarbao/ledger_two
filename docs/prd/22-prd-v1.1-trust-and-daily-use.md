## 1. 背景

LedgerTwo 已完成 v1.0/MVP 功能，但要成为真实长期使用的账本，还需要解决两个问题：

1. 用户敢不敢把真实账务长期放进来。
2. 用户愿不愿意每天快速记账。

v1.1 不追求大功能扩张，而是围绕可信赖、高频、清晰、可维护做产品收敛。

## 2. 目标

1. 普通支出 10 秒内完成。
2. 共同支出 20 秒内完成。
3. 结算页能让用户一眼看懂谁欠谁。
4. 分类、标签、账户能长期维护。
5. 移动端完成记账、筛选、结算。
6. 高风险操作都有明确确认。

## 3. 非目标

1. 不做直接通知共同支付。
2. 不做 OCR。
3. 不做银行同步。
4. 不做原生 App。
5. 不做公开邀请链接。
6. 不做复杂预算系统。
7. 不做企业级权限。

## 4. 用户故事

### 4.1 快速记录普通支出

作为日常使用者，我希望打开手机后快速输入金额、分类和备注，默认付款人和账户自动带出，这样我不用每次重复选择。

验收：

- 进入记账入口后默认显示最近使用配置。
- 用户只输入金额和分类即可保存。
- 保存成功后列表和 Dashboard 刷新。

### 4.2 快速记录共同支出

作为两人共同生活用户，我希望一笔共同支出默认两人参与、默认平均分摊，并能清楚看到双方承担金额。

验收：

- 共同支出默认参与人为当前账本成员。
- 默认分摊方式为 equal，可切换 payer_only。
- 预览显示每个成员 share amount。

### 4.3 复制一笔

作为长期用户，我希望复制常见账单，例如外卖、交通、房租，减少重复输入。

验收：

- 普通账单和共同支出都可复制。
- 复制时保留分类、账户、付款人、标签、分摊方式、备注。
- 发生时间默认当前时间。
- 复制不会修改原账单。

### 4.4 账单模板

作为有固定账单的用户，我希望保存常用模板，一键生成账单。

验收：

- 模板不进入统计和结算。
- 模板可生成普通账单和共同支出。
- 删除模板不影响历史账单。

### 4.5 周期账单提醒

作为有房租、水电、订阅费用的用户，我希望系统提醒我确认账单，而不是自动入账。

验收：

- 到期后生成待确认提醒。
- 用户确认后才生成真实账单。
- 取消提醒不影响历史账单。

### 4.6 分类、标签、账户管理

作为长期用户，我希望能维护分类、标签和账户，避免几年后账本变乱。

验收：

- 支持新增、编辑、排序、归档、恢复。
- 已归档项不出现在新增账单默认选择器。
- 历史账单仍显示已归档名称。
- 已使用项不物理删除。

### 4.7 结算文案复制

作为共同账本用户，我希望需要提醒对方转账时，能复制一段清晰文案到微信，而不是系统自动发通知。

验收：

- 结算页提供“复制结算文案”。
- 文案包含付款人、收款人、金额和月份。
- 不记录支付通知状态。

示例：

```text
2026-07 共同账本结算：lynn 应转给 polar ¥128.50。历史账单不会被修改，系统会生成结算记录。
```

## 5. 页面需求

### 5.1 记账表单

必须支持：

- 金额。
- 类型。
- 分类。
- 账户。
- 付款人。
- 参与人。
- 分摊方式。
- 标签。
- 时间。
- 备注。
- 可见性。

交互规则：

- 高频字段优先展示。
- 低频字段折叠。
- 移动端避免一屏过长，可分组。
- 离线时保存草稿，不提交正式账单。

### 5.2 流水页

必须支持：

- 月份筛选。
- 类型筛选。
- 成员筛选。
- 分类/标签筛选。
- 金额区间。
- 关键词。
- 复制一笔。
- 删除确认。

### 5.3 结算页

必须展示：

- 当前谁应给谁多少钱。
- 每个成员 paid/share/raw_net/settlement/final_net。
- 影响结算的共同支出明细入口。
- 已结算记录。
- 复制结算文案。

### 5.4 设置页

拆分为：

1. 账号与登录。
2. 账本与成员。
3. 分类、标签、支付账户。
4. 模板与周期账单。
5. 导入导出。
6. 备份恢复。
7. 系统诊断。

## 6. 权限需求

| 操作 | owner | editor | viewer |
|---|---:|---:|---:|
| 查看授权账单 | 是 | 是 | 是 |
| 新增账单 | 是 | 是 | 否 |
| 编辑自己创建账单 | 是 | 是 | 否 |
| 删除自己创建账单 | 是 | 是 | 否 |
| 管理分类标签账户 | 是 | 可配置，默认否 | 否 |
| 导出数据 | 是 | 否 | 否 |
| 备份恢复 | 是 | 否 | 否 |
| 管理成员 | 是 | 否 | 否 |

## 7. 数据与安全

1. 金额永远使用整数分。
2. 删除账单使用 soft delete。
3. 修改金额、删除、结算、备份、导出写审计日志。
4. private 账单和附件不可泄露。
5. settlement 不进入消费统计。

## 9. 风险

| 风险 | 应对 |
|---|---|
| 表单继续膨胀 | 高频/低频字段分层，必要时拆 Normal/Shared/Template 表单 |
| 分类标签管理破坏历史账单 | 使用归档，不物理删除已使用项 |
| 结算文案被误解为已支付 | 文案仅说明待结算，不改变系统状态 |
| 周期账单误自动入账 | 必须用户确认后生成真实账单 |
