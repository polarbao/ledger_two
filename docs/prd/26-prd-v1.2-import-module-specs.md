# PRD：v1.2 导入与省时间模块规格

状态：建议采纳  
目标版本：v1.2  
前置条件：v1.1 完成，分类/标签/账户管理稳定

## 1. 背景

长期记账的最大阻力之一是补账成本。v1.2 的目标不是“全自动记账”，而是提供可控、可预览、可回滚的 CSV 导入能力。

## 2. 目标

1. 用户可以导入微信/支付宝 CSV。
2. 上传后先预览，不直接写正式账单。
3. 用户能调整字段、分类、标签、账户。
4. 重复导入不会产生重复账单。
5. 批次提交失败时整体回滚。

## 3. 非目标

1. 不做银行同步。
2. 不做 OCR。
3. 不自动读取用户微信或支付宝账号。
4. 不绕过用户确认自动入账。
5. 不保证兼容所有历史 CSV 变体，优先支持当前主流导出格式。

## 4. 导入流程

```text
上传文件
-> 解析格式
-> 字段映射
-> 预览和错误提示
-> 去重分析
-> 规则推荐分类/标签/账户
-> 用户人工调整
-> 确认导入
-> 事务落库
-> 展示结果
```

## 5. CSV 导入预览

### 5.1 支持来源

短期支持：

- 微信支付 CSV。
- 支付宝账单 CSV。
- 通用 CSV 模板。

暂不支持：

- 银行 CSV。
- Excel xlsx。
- 图片或 PDF。

### 5.2 预览字段

标准预览字段：

- occurred_at。
- title。
- merchant。
- description。
- amount_cents。
- direction：expense/income/refund/unknown。
- source_account。
- suggested_category_id。
- suggested_tag_ids。
- suggested_account_id。
- duplicate_status。
- row_status。
- error_message。

### 5.3 验收

- 解析失败显示行号和原因。
- 金额转换为整数分。
- 用户确认前不写 transactions。

## 6. 去重与事务落库

### 6.1 去重规则

建议 import_hash 输入：

- source_type。
- occurred_at。
- amount_cents。
- merchant 或 title。
- external_order_id，如 CSV 存在。

去重状态：

| 状态 | 说明 |
|---|---|
| new | 可导入 |
| duplicate | 已存在，默认跳过 |
| suspicious | 疑似重复，需要用户确认 |
| invalid | 数据错误，不可导入 |

### 6.2 事务规则

- 同一批次确认导入必须在数据库事务内完成。
- 任一必需行失败时，整批回滚。
- 用户标记跳过的行不参与事务写入。
- 成功导入后记录 import_batch 和 import_row 状态。

### 6.3 验收

- 同一文件重复导入不会产生重复账单。
- 部分行错误时不会写入半批数据。
- 成功导入后可在批次详情看到导入数量、跳过数量、失败数量。

## 7. 导入规则

### 7.1 规则类型

支持：

- 商户包含关键词。
- 描述包含关键词。
- 金额区间，可选。
- 来源账户匹配，可选。

推荐结果：

- 分类。
- 标签。
- 支付账户。
- 可见性默认值。

### 7.2 规则优先级

1. 用户在预览页手工选择优先。
2. 精确商户规则优先。
3. 描述关键词规则次之。
4. 默认规则最后。

### 7.3 验收

- 规则命中结果在预览页可见。
- 用户手工修改后，不被规则再次覆盖。
- 禁用规则后不再命中。

## 8. 权限与安全

- 只有 owner 可执行导入。
- editor 默认不可导入，除非后续 RBAC 明确允许。
- viewer 不可导入。
- 导入文件不长期保留原文，除非用户明确选择保存。
- 导入批次只在当前账本可见。

## 9. 风险

| 风险 | 应对 |
|---|---|
| CSV 格式变化 | parser 分来源实现，错误提示给出可行动原因 |
| 重复判断误伤 | suspicious 状态交给用户确认 |
| 自动规则造成错误分类 | 规则只推荐，不自动提交 |
| 大文件导入卡顿 | 限制单批行数，必要时后台任务化 |
