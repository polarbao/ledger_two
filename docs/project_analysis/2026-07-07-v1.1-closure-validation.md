## 3. 未完成验收

以下事项仍是 v1.1 冻结前的必要收口项：

1. 浏览器内登录验证：确认 NAS 当前账号可登录，登录后 Dashboard 正常加载。
2. 记账路径验证：普通支出、共同支出、保存并继续记、复制一笔、模板生成账单。
3. 结算路径验证：结算页解释字段、影响明细入口、复制结算文案。
4. 设置路径验证：分类/标签/账户归档恢复、模板管理、周期账单确认/跳过。
5. 数据安全验证：附件上传与受控读取、手动备份、系统诊断。
6. 移动端截图或手工记录：375px、390px、430px 下覆盖 Dashboard、记账、流水筛选、结算、设置。

## 4. 收口策略

推荐下一步按以下顺序执行：

1. 使用真实 NAS 内测账号完成浏览器内业务流验收。
2. 只修复验收中发现的阻断缺陷，不再新增 v1.1 以外功能。
3. 若移动端只发现轻微排版问题，集中归入 Task46 收口；若出现业务流不可用，优先修复对应模块。
4. 浏览器内验收通过后，将 `docs/prd/22-prd-v1.1-trust-and-daily-use.md` 验收清单改为带证据的完成状态。

## 5. 阶段结论

当前 v1.1 处于“代码基本完成，部署已可内测，等待业务流验收冻结”的阶段。后续预计再经过 1 次以浏览器验收为主的会话，若没有发现阻断缺陷，可以完成 v1.1 冻结标记；若发现移动端或权限问题，预计需要 1-2 次修复会话。
