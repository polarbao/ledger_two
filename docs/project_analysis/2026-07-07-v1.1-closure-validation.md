- 本地地址：`http://localhost:38088`
- 容器状态：`ledger-two` 为 `healthy`
- 健康检查：`version=1.1.0-rc`、`db=ok`、`schema_version=12`
- 验收证据：`docs/project_analysis/v1.1-local-acceptance-2026-07-07/`
- 截图范围：登录页、Dashboard、记账抽屉、流水列表、移动端筛选 Sheet、结算中心、系统设置、分类管理
- 移动端宽度：`375px`、`390px`、`430px`
- 指标结论：14 个截图点，横向溢出 0，React Router 错误页 0

1. `/api/admin/backups` 无备份文件时返回 `data: null`，导致设置页读取 `backups.length` 崩溃；已改为后端返回空数组，前端额外做数组防御。
2. 移动端顶部栏 `desktop-user-info` 被组件内联 `display: flex` 覆盖媒体查询，导致用户信息挤压；已移除内联显示样式，将桌面布局回收到 CSS 类。

## 3. 未完成验收

以下事项仍是 v1.1 冻结前的必要收口项：

1. NAS 地址下登录验证：确认当前账号可登录，登录后 Dashboard 正常加载。
2. 记账路径深度验证：通过 UI 实际提交普通支出、共同支出、保存并继续记、复制一笔、模板生成账单。
3. 结算路径深度验证：登记结算、复制结算文案、结算历史记录和影响明细入口。
4. 设置路径深度验证：分类/标签/账户归档恢复、模板管理、周期账单确认/跳过。
5. 数据安全验证：附件上传与受控读取、手动备份创建/下载、系统诊断。
6. NAS 与本机部署一致性：本机验证通过后，同步确认 NAS 容器版本、schema 和关键入口截图。

## 4. 收口策略

推荐下一步按以下顺序执行：

1. 以本机 WSL2 作为快速验收环境，优先完成 UI 实际提交路径和数据安全路径。
2. 使用真实 NAS 内测账号完成同等浏览器内业务流验收。
3. 只修复验收中发现的阻断缺陷，不再新增 v1.1 以外功能。
4. 浏览器内深度验收通过后，将 `docs/prd/22-prd-v1.1-trust-and-daily-use.md` 验收清单改为带证据的完成状态。

## 5. 阶段结论

当前 v1.1 处于“代码基本完成，本机部署可验收，移动端关键入口通过，等待深度业务流和 NAS 复核冻结”的阶段。后续预计再经过 1 次以 UI 实际提交和数据安全为主的会话，若没有发现阻断缺陷，可以完成 v1.1 冻结标记；若发现权限、附件或备份恢复问题，预计需要 1-2 次修复会话。

补充：本机 WSL2 已完成本地部署验证，后续浏览器验收优先使用 `http://localhost:38088`，详见 `docs/project_analysis/2026-07-07-wsl2-local-deploy-and-v1.1-next.md`。
