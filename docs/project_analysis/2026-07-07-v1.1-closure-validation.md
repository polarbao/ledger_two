## 3. 未完成验收

以下事项仍是 v1.1 冻结前的必要收口项：

1. NAS 地址下登录验证：确认当前账号可登录，登录后 Dashboard 正常加载。

3. NAS 地址下数据安全复核：附件上传与受控读取、手动备份创建/下载、系统诊断。
4. NAS 与本机部署一致性：确认 NAS 容器版本、schema 和关键入口截图。

## 4. 收口策略

推荐下一步按以下顺序执行：

1. 以本机 WSL2 作为快速验收环境，优先完成 UI 实际提交路径和数据安全路径。
2. 使用真实 NAS 内测账号完成同等浏览器内业务流验收。
3. 只修复验收中发现的阻断缺陷，不再新增 v1.1 以外功能。
4. 浏览器内深度验收通过后，将 `docs/prd/22-prd-v1.1-trust-and-daily-use.md` 验收清单改为带证据的完成状态。

## 5. 阶段结论

当前 v1.1 处于“代码基本完成，本机部署可验收，移动端关键入口通过，核心记账、复制模板、结算路径、设置路径和数据安全 API 验收通过，等待 NAS 复核冻结”的阶段。后续预计再经过 1 次以 NAS 复核为主的会话，若没有发现阻断缺陷，可以完成 v1.1 冻结标记；若发现 NAS 环境权限、附件或备份恢复问题，预计需要 1 次修复会话。
