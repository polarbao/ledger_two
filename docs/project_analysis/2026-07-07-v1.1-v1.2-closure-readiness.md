# LedgerTwo v1.1 收口与 v1.2 文档就绪判断

日期：2026-07-07

## 1. 判断结论

### 1.1 v1.1 是否可以继续收口

可以继续收口，但不建议直接冻结。

理由：

1. 代码层面 Task41-Task46 的主体功能已有落点。
2. 本机 WSL2 部署可用，`/api/healthz` 返回 `version=1.1.0-rc`、`schema_version=12`、`db=ok`。
3. 移动端关键入口已完成 Chrome CDP 截图验收，覆盖 375px/390px/430px，横向溢出 0，React Router 错误页 0。
4. 普通支出保存并继续、共同支出提交、equal split 和结算余额联动已完成本机 WSL2 UI 实际提交验收。
5. 结算页复制文案兜底、登记结算、结算历史和 375px 移动端无溢出已完成本机 WSL2 UI 实际验收。
6. 复制一笔、存为模板、模板填入和模板生成账单已完成本机 WSL2 UI 实际验收。
7. 本轮验收发现的设置页空备份列表崩溃、移动端顶栏挤压、保存并继续成功后误关闭抽屉、流水卡片展示 category_id、结算文案复制失败无兜底、流水卡片长标题挤压金额已修复。

不能直接冻结的原因：

1. 附件上传与受控读取、手动备份创建/下载、恢复前二次确认仍需深度验收。
2. 分类/标签/账户归档恢复、模板管理、周期账单确认/跳过仍需深度验收。
3. NAS 地址下需要复核版本、schema、登录和关键入口。
4. v1.1 PRD 的验收清单尚未全部改成带证据的完成状态。

## 3. v1.1 后续收口建议

下一步建议不再扩大 v1.1 功能，只做验收和阻断缺陷修复：

1. 验证附件上传与受控读取。
2. 验证手动备份创建、下载、恢复前确认。
3. 验证分类/标签/账户归档恢复、模板管理、周期账单确认/跳过。
4. 在 NAS 地址复核 healthz、登录、Dashboard、设置页和关键操作入口。
5. 更新 `docs/prd/22-prd-v1.1-trust-and-daily-use.md` 的验收状态。

## 4. v1.2 进入开发前检查

Task47 启动前必须确认：

1. v1.1 已冻结或明确不阻塞 v1.2 的剩余问题。
2. `docs/tech/20-v1.2-import-implementation-contract.md` 中 open questions 至少对 Task47 不构成阻塞。
3. API Inventory 和 OpenAPI 准备随 Task47 更新。
4. migration 不修改历史文件，只新增。
5. 微信、支付宝、通用 CSV fixture 准备完成。
6. 导入提交入口默认可关闭，preview 可独立运行。

## 5. 阶段判断

当前项目处于：

```text
v1.1 收口验收后半段
-> 完成设置深度验证、数据安全和 NAS 复核
-> v1.1 冻结
-> Task47 v1.2 CSV 预览开发
```

预计：

- v1.1 若无阻断缺陷，仍需 1 次以设置深度验证、数据安全和 NAS 复核为主的会话完成冻结。
- v1.2 文档已经可以支持开发前评审和 Task47 拆分；正式开发时仍需每个 task 随代码补 OpenAPI、migration 和测试 fixture。
