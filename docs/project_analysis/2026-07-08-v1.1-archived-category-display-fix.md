# LedgerTwo v1.1 历史归档分类展示收口记录

日期：2026-07-08  
阶段：v1.1 冻结前  
对应 PRD：`docs/prd/22-prd-v1.1-trust-and-daily-use.md`

## 1. 背景

v1.1 PRD 要求：

```text
历史账单仍显示已归档名称。
```

此前已完成分类、标签、账户归档恢复 API 验收，但流水页分类展示仍依赖 `/api/categories` 的未归档分类列表。

风险：

1. 历史账单引用已归档分类时，前端可能无法从分类映射中找到名称。
2. 移动端流水卡片会退化展示为“已设分类”，不满足“显示已归档名称”的 PRD 要求。

## 2. 本轮改造

### 2.1 后端

改造文件：

```text
backend/internal/transaction/handler.go
backend/internal/transaction/service.go
backend/internal/transaction/repo.go
```

变更：

1. `/api/categories` 支持 `include_archived=true`。
2. 默认 `/api/categories` 仍只返回未归档分类，保证新增账单选择器不展示归档项。
3. 分类 DTO 增加 `is_archived` 字段。

### 2.2 前端

改造文件：

```text
frontend/src/api/transactions.api.ts
frontend/src/api/queryKeys.ts
frontend/src/types/transaction.ts
frontend/src/pages/TransactionsPage.tsx
```

变更：

1. `transactionsApi.getCategories({ includeArchived: true })` 支持历史展示场景。
2. `TransactionsPage` 使用包含归档项的分类映射。
3. 已归档分类展示为 `分类名（已归档）`。
4. 记账表单、周期规则、旧导入页仍使用默认未归档分类列表。

## 3. 回归测试补充

改造文件：

```text
backend/internal/http/handler/transaction_test.go
```

新增断言：

1. 创建一笔引用分类的历史账单。
2. 将该分类归档。
3. 默认 `/api/categories` 不返回该分类。
4. `/api/categories?include_archived=true` 返回该分类名称和 `is_archived=true`。
