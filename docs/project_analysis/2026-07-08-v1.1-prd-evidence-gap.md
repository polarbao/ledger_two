# LedgerTwo v1.1 PRD 剩余证据缺口

日期：2026-07-08  
阶段：v1.1 冻结前

## 1. 结论

当前 v1.1 可以继续收口，但还不应仅凭已有记录直接冻结。

原因不是核心功能缺失，而是 PRD 中仍有少数验收项需要更严格证据：

1. 普通支出 10 秒内完成。
2. 共同支出 20 秒内完成。
3. 历史账单保留归档项展示。
4. NAS 地址下登录和关键业务路径 UI 复核。

## 3. 不应直接勾选的项目

### 3.1 普通支出 10 秒内完成

已有真实 UI 提交成功，但验收记录没有记录从打开抽屉到提交成功的耗时。

冻结前建议补：

1. 375px 移动端。
2. 记一笔入口打开。
3. 只输入金额、分类或标题的最短路径。
4. 记录开始时间、点击保存时间、成功状态时间。

通过标准：

```text
submit_success_at - drawer_open_at <= 10s
```

### 3.2 共同支出 20 秒内完成

已有真实 UI 提交成功和 equal split 证据，但没有严格耗时。

冻结前建议补：

```text
submit_success_at - drawer_open_at <= 20s
```

同时确认：

1. 默认参与人为当前账本成员。
2. 默认分摊方式为 equal。
3. 提交前可见双方承担金额。

### 3.3 历史账单保留归档项展示

已有归档/恢复 API 证据，但缺少“创建账单引用分类/账户 -> 归档该分类/账户 -> 历史账单仍展示名称”的端到端证据。

冻结前建议补：

1. 新建分类或账户。
2. 创建一笔引用该元数据的账单。
3. 归档该元数据。
4. 打开流水列表和详情。
5. 验证展示用户可读名称，并带已归档提示或至少不退化成 UUID。

## 4. NAS UI 复核清单

NAS 地址：

```text
http://192.168.0.115:38088
```

只读健康检查已通过，仍需 UI 级复核：

1. 登录页可登录。
2. Dashboard 正常加载。
3. 流水列表和记账抽屉正常。
4. 结算中心可展示和复制文案。
5. 设置页可进入分类、标签、账户管理。
6. 附件上传和受控读取正常。
7. 手动备份创建和下载正常。
8. 恢复备份弹窗只做准备确认，不在未确认情况下执行恢复。
