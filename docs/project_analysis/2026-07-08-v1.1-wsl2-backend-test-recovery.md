# LedgerTwo v1.1 WSL2 后端测试环境与测试债收口

日期：2026-07-08  
阶段：v1.1 冻结前

## 1. 背景

此前 Windows 原生执行后端测试被 `go-sqlite3` cgo 对象解析问题阻断：

```text
cgo: cannot parse gcc output ... as ELF, Mach-O, PE, XCOFF object
```

WSL2 中已有 `gcc` 和 `sqlite3`，但缺少 Go，因此无法在 WSL2 中补跑后端测试。

## 2. 本轮环境补齐

已在 WSL2 中安装：

```bash
sudo apt-get update
sudo apt-get install -y golang-go
```

安装后版本：

```text
go version go1.26.4 linux/amd64
```

已有环境：

```text
gcc: installed
sqlite3: installed
```

## 3. 本轮测试债修复

改造文件：

```text
backend/internal/http/handler/attachment_test.go
backend/internal/http/handler/safety_test.go
backend/internal/http/handler/template_test.go
backend/internal/http/handler/transaction_test.go
```

修复内容：

1. 多个 handler 测试使用 `userA` 字符串作为 `payer_user_id`，已改为初始化后查询真实 `users.id`。
2. `safety_test` 未挂载真实 router 中的 `ledger.WithLedgerContext`，已补齐。
3. `safety_test` 的备份、导出请求已显式传入 `X-Ledger-Id`，符合当前 ledger-context 约束。

判断：

这些是测试夹具落后于 v1.1 RBAC/LedgerContext 收口，不是业务代码缺陷。
