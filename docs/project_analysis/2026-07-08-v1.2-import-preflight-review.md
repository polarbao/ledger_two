# LedgerTwo v1.2 导入模块开工前评审

日期：2026-07-08  
阶段：v1.2 Task47-Task49 准备

## 1. 结论

v1.2 可以继续做准备性工作，但正式 Task47 代码开发仍建议等待 v1.1 冻结或明确豁免。

本轮已补：

1. 导入 fixture：`docs/fixtures/imports/`
2. OpenAPI 草案：`docs/api/openapi-v1.2-import-draft.yaml`
3. Migration 评审：`docs/tech/21-v1.2-import-migration-review.md`
4. Task47 实施计划：`docs/tech/22-v1.2-import-task47-implementation-plan.md`

## 2. 设计评审

当前 v1.2 UI 设计方向已足够支持 Task47：

1. `docs/ui/figma/v1.1-v1.2-ui-draft-spec.md` 已覆盖 Import Entry、Preview Desktop、Preview Mobile、Row Editor、Commit Confirm、Rule Manager。
2. `docs/ui/figma/ledger-two-frame-manifest.json` 已给出 Figma frame 清单。
3. `docs/ui/figma/component-library.md` 已覆盖 Import Row Card、Commit Confirm Modal 和 Import Rule Manager。

评审结论：

1. Task47 先做 Import Entry 和 Preview Workbench。
2. 移动端 preview 必须用卡片，不上宽表格。
3. Commit Confirm 属于 Task48，不应在 Task47 提前开放。
4. Import Rule Manager 属于 Task49，不应混入 Task47。
5. UI/UX 任务应作为 `Task47U/48U/49U` 并行切片进入开发计划，而不是作为 Task49 后的美化收尾。

补充决策：

```text
Task47U = 导入入口与预览工作台
Task48U = 提交确认与结果反馈
Task49U = 规则管理与推荐解释
```

原因：

1. 导入模块的核心风险不只是 parser，而是用户是否能理解“预览未入账、疑似重复需处理、提交会写入多少行”。
2. 移动端导入预览如果后置，很容易在 Task47 完成后才发现宽表格不可用。
3. 规则推荐如果缺少解释，容易被用户误解为自动分类，违背“规则只推荐、不自动提交”的 v1.2 原则。

补充：

1. `docs/fixtures/imports/expected/*.preview.json` 已给出 parser/preview 的预期输出。
2. Task47 parser 测试必须以 expected JSON 作为断言基线，避免只验证“能解析”而没有验证金额、方向、跳过和 invalid 行语义。

## 4. OpenAPI 评审

新增草案：

```text
docs/api/openapi-v1.2-import-draft.yaml
```

原因：

1. `docs/api/openapi.yaml` 描述当前已实现 API，不应把未实现 `/api/imports/*` 写成已上线接口。
2. v1.2 需要前后端先对齐目标契约。
3. 旧 `/api/transactions/import/*` 仍是 transitional，不作为新前端目标。

Task47 目标接口：

```text
POST /api/imports/preview
GET  /api/imports/{batch_id}
PATCH /api/imports/{batch_id}/rows/{row_id}
```

Task48 目标接口：

```text
POST /api/imports/{batch_id}/commit
```

Task49 目标接口：

```text
GET   /api/import-rules
POST  /api/import-rules
PATCH /api/import-rules/{rule_id}
POST  /api/import-rules/{rule_id}/archive
POST  /api/import-rules/{rule_id}/restore
```

## 5. Migration 评审

当前 `004_add_import_tables.sql` 和 `005_add_import_rules.sql` 是早期导入实现，不能修改。

建议新增：

1. `013_add_import_preview_schema.sql`
2. `014_add_transaction_import_refs.sql`
3. `015_extend_import_rules_for_v12.sql`

关键决策：

1. 新增 `import_rows`，不把旧 `import_items` 硬改成预览行。
2. Task48 使用 `transaction_import_refs` 作为正式去重唯一映射，暂不改 `transactions` 主表。
3. Task49 扩展 `import_rules`，旧 keyword 规则可兼容迁移。

## 6. Task47 开工清单

可以开工前需要确认：

1. v1.1 冻结或豁免。
2. `commit` 开关在 Task47 阶段关闭。
3. owner-only 权限作为后端强约束。
4. parser fixture 测试先覆盖微信、支付宝、通用 CSV。

6. 前端 query key 必须包含 ledgerId。
7. Task47U 与后端 Task47 并行启动，预览工作台必须在 375px 上可用。
