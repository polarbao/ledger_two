## 3. 冻结边界

v1.2 不新增：

1. OCR。
2. 银行同步。
3. 自动提交 suspicious 行。
4. editor 导入权限。
5. 直接通知共同支付。
6. 导入批次撤销。

旧 `/api/transactions/import/*` 和 `DELETE /api/import-rules/{ruleID}` 继续作为 transitional 兼容入口，不作为新前端开发入口。

## 4. 非阻断风险

1. 前端主 chunk 约 652 kB，Vite 继续提示大包警告；进入后续性能专项，不阻断 v1.2 内测。
2. Figma 当前主文件覆盖 Task49 核心设计，不等同于 LedgerTwo 全版本逐层完整设计系统。
3. 本机 QA 数据包含历史验收元数据，仅用于本地验证，不应同步为生产初始数据。
4. NAS 更新未在本轮执行，保持此前“本地大版本验收后再同步 NAS”的部署策略。

## 5. 下一步

1. 保持业务代码冻结，只接受阻断级缺陷修复。
2. 生成 v1.2 发布说明和升级说明。
3. 在确认 NAS 更新窗口后同步 `1.2.0-rc`，复核 health、登录、导入预览和备份。
4. v1.3 开工前重新评审多账本、多成员和多人分摊范围，不直接沿用早期规划开工。
