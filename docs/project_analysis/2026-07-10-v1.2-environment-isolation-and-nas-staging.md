# v1.2 环境隔离与 NAS Staging 准备记录

日期：2026-07-10

## 1. 决策结论

NAS 必须区分 staging 与 production，不能让候选版本直接覆盖唯一正式实例。v1.2 将该工作归入 Release Hardening RC02/RC03，而不是继续扩展 Task47-Task49 业务范围。

隔离方式：

1. production：`/volume1/docker/ledger-two`、端口 `38088`、channel `production`、原数据库。
2. staging：`/volume1/docker/ledger-two-staging`、端口 `38089`、channel `staging`、生产备份副本。
3. 两者使用不同 Compose project、容器名、JWT secret、data/backups/uploads/logs。
4. staging 数据只用于迁移和验收，不回写 production。

4. 失败时保存故障数据库、恢复最终 schema 12 备份并启动旧容器。
