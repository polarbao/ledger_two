## 4. 切换与回滚保护

直接 Compose recreate 因旧 project 标签未替换运行容器，期间 production 始终保持 v1.1/schema 12，没有半升级。最终使用 `deploy/nas/promote-production.sh`：

1. 停止并保留旧 v1.1 容器。
2. 使用独立 production project 启动候选镜像。
3. 自动校验 version/schema/channel/db。
4. 失败时恢复最终 schema 12 数据库和旧容器。

本次自动 health 门禁通过，未触发回滚。旧容器暂时保留，稳定观察结束前不删除；旧容器不得直接连接 schema 18 数据库，回滚时必须同时恢复 schema 12 最终备份。
