# 人类审核清单

## 1. 通用审核

- [ ] 任务是否只实现了当前范围？
- [ ] 是否实现了未审核 v1.1 业务？如果是，应退回。
- [ ] 是否有测试命令和结果？
- [ ] 是否修改了历史 migration？
- [ ] 是否提交了真实密钥、数据库、备份或上传文件？
- [ ] 是否破坏 v1.0 已有核心功能？

## 2. 后端审核

- [ ] 金额是否全部使用 int64 cents？
- [ ] handler 是否只处理 HTTP？
- [ ] service 是否包含业务规则？
- [ ] repository 是否只做数据库访问？
- [ ] 所有 ledger scoped 查询是否带 ledger_id？
- [ ] 权限是否在后端校验？
- [ ] private 数据是否可能泄露到列表、详情、导出、附件？
- [ ] 高风险操作是否写审计日志？

## 3. 前端审核

- [ ] query key 是否包含 ledgerId？
- [ ] 切换账本是否不再强制 reload？
- [ ] 是否覆盖 loading/empty/error/forbidden/offline？
- [ ] 是否使用 PermissionGate 控制角色体验？
- [ ] 表单是否使用 Zod 校验？
- [ ] 金额是否提交 cents？
- [ ] 移动端是否无横向滚动？

## 4. 文档审核

- [ ] README、PRD、Tech、UI、Task 文档状态是否一致？
- [ ] 是否明确 v1.1 尚未审核？
- [ ] 是否有不做事项？
- [ ] 是否有验收标准？
- [ ] 是否给 Codex/Gemini 足够明确的输入文档？

## 5. 发布审核

- [ ] Docker build 是否通过？
- [ ] 配置示例是否无真实密钥？
- [ ] production 缺少密钥是否启动失败？
- [ ] 备份恢复说明是否清晰？
- [ ] 迁移失败是否有回滚方案？
