# PRD 模块目录（Post-Task30 Foundation）

本目录用于描述 LedgerTwo 当前产品状态、后续路线和进入 v1.1 前必须补足的基础框架要求。

## 1. 当前阶段

Task01-Task30 已完成，项目进入 v1.0 后的基础框架补齐阶段。v1.1 具体业务规划尚未审核定稿，因此本目录当前重点不是描述 v1.1 完整业务，而是对齐 MVP/v1.0 已实现能力和补齐后续开发所需基础框架。

## 4. PRD 编写原则

1. 区分“已实现能力”“文档规划能力”“待审核能力”。
2. 待审核的 v1.1 业务能力不得写成已确认开发任务。
3. 基础框架能力可以先开发，但必须不破坏 v1.0 已完成能力。
4. 所有需求必须有验收标准和不做事项。
5. AI 开发任务必须能从 PRD 直接追溯到 Tech/UI/Test 文档。
