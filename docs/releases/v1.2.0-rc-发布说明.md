# LedgerTwo v1.2.0-rc 发布说明

发布日期：2026-07-10

发布状态：Release Candidate，仅用于本机与 NAS 内测

数据库版本：`18`

范围更新（2026-07-14）：本机候选已实现 Task49X、运行开关并升级到 schema 19，微信 XLSX 276 行与支付宝 CSV 1232 行均完成 preview，未提交正式账单。支付宝当前仍只导出 CSV，前后端均拒绝支付宝 XLSX。NAS 当前仍是 schema 18 基线，不得把本机能力视为 NAS 已发布；剩余发布门禁为 NAS schema 19 staging、production 一致性备份与逐批导入确认。

## 1. 版本定位

v1.2 聚焦“让账单进入系统之前可预览、可解释、可纠错”，并把 v1.1 的日常记账、模板、周期账单、元数据管理、结算解释和移动端体验一并纳入候选基线。该版本不继续扩张到 OCR、银行同步或自动通知共同支付。

## 2. 主要变化

### 2.1 日常记账与可信基础

1. 快捷记账默认值、保存并继续记和移动端金额输入优化。
2. 账单复制、从已有账单保存模板、模板编辑/归档/恢复和实例化重校验。
3. 周期账单先生成待确认提醒，确认后才写正式账单，支持跳过本期。
4. 分类、标签、支付账户支持排序、归档、恢复；历史账单保留归档项显示。
5. 结算页展示 paid、share、raw_net、settlement、final_net，并支持复制结算文案。
6. LedgerContext/RBAC、受保护附件访问、审计、系统诊断和生产配置校验进入冻结基线。

### 2.2 CSV 导入

1. 支持微信、支付宝和通用 CSV 的预览解析；预览不会写入正式账单。
2. 支持行级分类、账户、标签、可见性和目标类型调整。
3. 支持 duplicate、suspicious、invalid/new 状态，重复默认跳过，疑似项必须人工确认。
4. 提交在单一数据库事务内完成；任一必需写入失败时整批回滚并反馈失败状态。
5. 使用 `transaction_import_refs` 在账本内按 `import_hash` 去重。

### 2.3 导入规则

1. Owner 可创建、编辑、归档和恢复关键词规则，并设置优先级与金额范围。
2. 规则可以推荐分类、账户和多个标签，预览行展示命中规则和解释。
3. 用户手工选择始终优先于规则建议。
4. 已归档规则以及引用已归档分类、账户或标签的规则不参与推荐。

## 3. 权限边界

1. v1.2 导入预览、批次读取、提交和规则管理均为 Owner-only。
2. Editor/Viewer 的前端入口与后端 API 都不得绕过该限制。
3. 私密账单、附件和账本隔离继续沿用现有后端可信边界。

## 4. 明确不包含

1. OCR 和银行自动同步。
2. 自动提交 suspicious 行。
3. 导入批次撤销或反向删除已生成账单。
4. 直接通知共同支付。
5. v1.3 多账本正式化、多人分摊增强和原生移动端应用。

## 5. 兼容性与已知风险

1. 旧 `/api/transactions/import/*` 与 `DELETE /api/import-rules/{ruleID}` 暂时保留兼容，新前端只使用 `/api/imports/*` 与 archive/restore。
2. 前端主 chunk 约 652 kB，构建存在大包提示但不阻断内测；后续进入性能专项。
3. Figma 主文件已覆盖 Task49 核心设计，但尚不是全产品逐屏完整设计系统。
4. 本机 QA 数据不是生产初始数据，NAS 升级不得覆盖为本机测试数据库。
