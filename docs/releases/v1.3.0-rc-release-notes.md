# LedgerTwo v1.3.0-rc 发布说明

发布日期：2026-07-17  
状态：本机独立 staging 候选已通过，NAS 未部署

## 主要变化

1. 支持同一用户创建、加入和切换多个账本。
2. 账本具备 active/archived 生命周期、版本化并发控制和归档只读历史。
3. 账本角色正式收敛为 Owner、Editor、Viewer；Task50 每账本仍最多两名成员。
4. 支持成员添加、角色调整、唯一 Owner 原子移交、移除和主动离开，历史账务对象不改写。
5. 实例管理员与账本 Owner 权限分离；整库诊断、备份和恢复准备不再依赖账本角色。
6. 所有业务 API 使用显式账本上下文，交易、结算、报表、元数据、导入、附件、导出和审计均按账本隔离。
7. JSON 导出升级为带 manifest 的只读账本数据包，覆盖模板、周期和导入引用；它不能替代物理备份。
8. Fresh Light 和 Dark Glass 在 375/390/430/1440 视口完成登录、流水、导入、结算、设置和账本管理回归。

## 数据库

目标 schema 为 21：

- migration 020：账本生命周期、version、唯一 Owner 索引、最多两人 trigger 和 ledger audit role。
- migration 021：实例管理员和实例审计。

从 schema 19 升级前会执行 quick_check、外键、成员数、唯一 Owner 和数据守恒预检。升级不会删除交易、结算、导入引用或附件引用。

## 范围边界

- 不开放第三名成员；多人分摊归 Task51 重新评审。
- 不实现支付通知、银行同步、OCR、企业租户或原生 App。
- 支付宝继续使用 CSV；微信支持 CSV/XLSX。
- NAS 部署需要单独维护窗口，不随本机候选自动执行。
