# 技术：测试与质量保障

## 1. 目标

测试与质量保障模块用于保证账务计算、权限、结算和数据安全在长期迭代中不被破坏。

## 2. 测试分层

### 2.1 后端单元测试

重点覆盖：

- 金额转换。
- 分摊算法。
- 结算算法。
- 权限判断。
- soft delete。
- 备份导出。

### 2.2 后端集成测试

使用测试 SQLite 数据库，覆盖：

- 初始化。
- 登录。
- 账单 CRUD。
- 共同支出。
- 结算。
- 统计。

### 2.3 前端测试

优先覆盖：

- 金额元/分转换。
- API client 错误处理。
- 表单校验。
- 关键页面渲染。

### 2.4 E2E 测试

后续使用 Playwright 覆盖核心路径：

1. 初始化两个用户。
2. 登录。
3. 创建共同支出。
4. 查看结算关系。
5. 生成结算记录。
6. 导出数据。

## 3. 必测业务用例

- A 支付 20000，两人平摊，B 欠 A 10000。
- B 支付 8000，两人平摊，合并后 B 欠 A 6000。
- B 结算 6000 后双方结清。
- private 账单对方不可见。
- partner_readable 账单对方可见但不可编辑。
- 删除账单后列表和统计不再出现。
- settlement 不进入消费统计。
- 金额为 0 或负数时创建失败。

## 4. 质量门禁

提交前建议执行：

```bash
cd backend && go test ./...
cd frontend && pnpm lint && pnpm build
```

## 5. CI 建议

GitHub Actions 后续增加：

- backend-test。
- frontend-lint-build。
- docker-build。
- markdown-link-check，可选。

## 6. 代码审查重点

- 是否出现 float 金额计算。
- 是否绕过 service 层直接操作数据库。
- 是否前端自行计算最终结算金额。
- 是否查询时遗漏 status != deleted。
- 是否 private 账单泄露。
- 是否把 SESSION_SECRET 或真实数据库提交到仓库。

## 7. 验收标准

- 核心结算测试稳定通过。
- 权限测试稳定通过。
- Docker 构建通过。
- 文档与代码实际功能保持一致。
