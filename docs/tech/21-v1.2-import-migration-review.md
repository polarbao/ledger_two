# 技术：v1.2 导入模块 Migration 评审

状态：建议采纳  
适用阶段：Task47 开工前  
关联文档：

- `docs/tech/20-v1.2-import-implementation-contract.md`
- `docs/prd/29-prd-v1.2-module-business-service-breakdown.md`
- `docs/api/openapi-v1.2-import-draft.yaml`
- `docs/fixtures/imports/README.md`

## 1. 结论

v1.2 不应修改历史 migration `004_add_import_tables.sql` 和 `005_add_import_rules.sql`。

建议从新增 migration `013_*` 开始演进导入模块：

1. Task47：扩展批次表，新增 `import_rows`，保留旧 `import_items`。
2. Task48：新增正式账单导入去重映射表，不把最终唯一约束压到 `transactions` 主表。
3. Task49：扩展 `import_rules` 为规则管理模型，支持 status、priority 和 result_json。

原因：

1. 历史 migration 已应用于本机和 NAS，不能回写修改。
2. 旧 `import_batches/import_items` 只支持“已导入/跳过”结果，不支持 preview row 状态机。
3. 直接复用旧 `import_items` 会把 Task47 预览态和 Task48 提交态混在一起。
4. 新增表比重命名旧表更容易回滚和兼容。

## 2. 当前 schema 评审

### 2.1 `import_batches`

当前字段：

```text
id
ledger_id
filename
created_by_user_id
status default completed
created_at
```

缺口：

1. 缺 `source_type`。
2. 缺 `file_sha256`。
3. 缺 ready/failed/expired/committed 状态机。
4. 缺行统计。
5. 缺 updated_at、committed_at、expires_at。

建议：

Task47 新增 nullable/default 字段，保留旧字段兼容。

### 2.2 `import_items`

当前字段：

```text
id
batch_id
transaction_id
import_hash
status imported/skipped
created_at
```

判断：

不适合作为 v1.2 preview row。

建议：

1. 保留为旧接口提交结果兼容表。
2. v1.2 新增 `import_rows`。
3. Task48 后旧接口若委托新 importer，可按需写兼容记录。

### 2.3 `import_rules`

当前字段：

```text
id
ledger_id
keyword
category_id
tag_names
account_id
created_by_user_id
created_at
updated_at
```

缺口：

1. 缺 match_type。
2. 缺 pattern。
3. 缺 amount_min_cents/amount_max_cents。
4. 缺 priority。
5. 缺 status/archived_at。
6. 缺 result_json。

建议：

Task49 用新增字段兼容旧 keyword 规则，不物理删除旧字段。

## 3. 推荐 migration 切片

### 3.1 Task47 migration：preview batch 与 row

建议文件：

```text
013_add_import_preview_schema.sql
```

建议内容：

```sql
ALTER TABLE import_batches ADD COLUMN source_type TEXT;
ALTER TABLE import_batches ADD COLUMN file_sha256 TEXT;
ALTER TABLE import_batches ADD COLUMN total_rows INTEGER NOT NULL DEFAULT 0;
ALTER TABLE import_batches ADD COLUMN new_rows INTEGER NOT NULL DEFAULT 0;
ALTER TABLE import_batches ADD COLUMN duplicate_rows INTEGER NOT NULL DEFAULT 0;
ALTER TABLE import_batches ADD COLUMN suspicious_rows INTEGER NOT NULL DEFAULT 0;
ALTER TABLE import_batches ADD COLUMN invalid_rows INTEGER NOT NULL DEFAULT 0;
ALTER TABLE import_batches ADD COLUMN imported_rows INTEGER NOT NULL DEFAULT 0;
ALTER TABLE import_batches ADD COLUMN skipped_rows INTEGER NOT NULL DEFAULT 0;
ALTER TABLE import_batches ADD COLUMN failed_rows INTEGER NOT NULL DEFAULT 0;
ALTER TABLE import_batches ADD COLUMN updated_at TEXT;
ALTER TABLE import_batches ADD COLUMN committed_at TEXT;
ALTER TABLE import_batches ADD COLUMN expires_at TEXT;

CREATE TABLE import_rows (...);
CREATE INDEX idx_import_rows_batch_row ON import_rows(batch_id, row_number);
CREATE INDEX idx_import_rows_ledger_hash ON import_rows(ledger_id, import_hash);
CREATE INDEX idx_import_rows_batch_duplicate ON import_rows(batch_id, duplicate_status);
CREATE INDEX idx_import_rows_batch_status ON import_rows(batch_id, row_status);
```

注意：

1. SQLite `ALTER TABLE ADD COLUMN` 不适合一次性加复杂 CHECK 约束，约束放 service 层和测试。
2. `status` 当前默认 `completed`，新 service 读取时需兼容旧值。
3. `import_rows.raw_json/normalized_json/user_adjustment_json` 使用 TEXT 存 JSON。

### 3.2 Task48 migration：正式去重映射

实际文件：

```text
015_add_transaction_import_refs.sql
```

使用独立映射表：

```sql
CREATE TABLE transaction_import_refs (
    id TEXT PRIMARY KEY,
    ledger_id TEXT NOT NULL,
    transaction_id TEXT NOT NULL,
    import_batch_id TEXT NOT NULL,
    import_row_id TEXT NOT NULL,
    import_hash TEXT NOT NULL,
    external_order_id TEXT,
    source_type TEXT NOT NULL,
    created_at TEXT NOT NULL,
    FOREIGN KEY (ledger_id) REFERENCES ledgers(id) ON DELETE CASCADE,
    FOREIGN KEY (transaction_id) REFERENCES transactions(id) ON DELETE CASCADE,
    FOREIGN KEY (import_batch_id) REFERENCES import_batches(id) ON DELETE CASCADE,
    FOREIGN KEY (import_row_id) REFERENCES import_rows(id) ON DELETE CASCADE
);
CREATE UNIQUE INDEX idx_transaction_import_refs_hash ON transaction_import_refs(ledger_id, import_hash);
CREATE INDEX idx_transaction_import_refs_tx ON transaction_import_refs(transaction_id);
```

理由：

1. 避免在 v1.2 初期修改核心 `transactions` 表。
2. 便于回滚导入模块，而不影响既有记账主表。
3. 后续如需查询性能，再把冗余字段同步到 `transactions`。

实现说明：

1. 由于 Task47 已使用 `013_extend_import_preview_tables.sql` 和 `014_add_import_row_selection_fields.sql` 扩展旧 `import_batches/import_items`，Task48 去重映射实际顺延为 `015_add_transaction_import_refs.sql`。
2. `transaction_import_refs.import_row_id` 当前引用扩展后的 `import_items.id`，其业务语义等价于实施契约中的 `import_rows.id`。
3. `transaction_import_refs(ledger_id, import_hash)` 是 Task48 冻结前的最终唯一约束。

### 3.3 Task49 migration：规则扩展

实际文件：

```text
017_extend_import_rules_for_v12.sql
```

建议内容：

```sql
ALTER TABLE import_rules ADD COLUMN name TEXT;
ALTER TABLE import_rules ADD COLUMN match_type TEXT;
ALTER TABLE import_rules ADD COLUMN pattern TEXT;
ALTER TABLE import_rules ADD COLUMN amount_min_cents INTEGER;
ALTER TABLE import_rules ADD COLUMN amount_max_cents INTEGER;
ALTER TABLE import_rules ADD COLUMN priority INTEGER NOT NULL DEFAULT 100;
ALTER TABLE import_rules ADD COLUMN result_json TEXT;
ALTER TABLE import_rules ADD COLUMN status TEXT NOT NULL DEFAULT 'active';
ALTER TABLE import_rules ADD COLUMN archived_at TEXT;
CREATE INDEX idx_import_rules_ledger_status_priority ON import_rules(ledger_id, status, priority);
```

实现说明：

1. Task48 已占用 `015_add_transaction_import_refs.sql` 和 `016_add_import_batch_commit_fields.sql`，因此 Task49 规则扩展实际顺延为 `017_extend_import_rules_for_v12.sql`。
2. 旧 `keyword/category_id/account_id/tag_names` 字段继续保留用于兼容读取，新增 `match_type/pattern/result_json/status/priority` 承载 v1.2 规则模型。
3. 旧数据在 migration 中按 `merchant_contains` 和 `visibility=private` 转换为 `result_json`，后续服务层以新字段为主。

兼容：

1. 旧 `keyword` 可迁移为 `match_type=merchant_contains`、`pattern=keyword`。
2. 旧 category/account/tag 字段可生成 `result_json`。
3. 旧 `DELETE /api/import-rules/{id}` 转为 archive。

## 4. 评审风险

| 风险 | 结论 |
|---|---|
| 旧 `status=completed` 与新状态机冲突 | 新 service 只处理 previewing/ready/committed/failed/expired；旧 completed 只读兼容 |
| import_hash 唯一约束误伤 | Task48 前先用 preview 标记 duplicate/suspicious，commit 才写唯一映射 |
| result_json 引用归档元数据 | Task49 service 应校验，不在 DB 里做复杂约束 |
| 大 CSV 写入卡顿 | Task47 限制 max_rows_per_batch，暂不做异步队列 |

## 5. 开工门槛

Task47 开工前必须确认：

1. v1.1 已冻结或剩余验收项被明确豁免。
2. 本文 migration 切片未与其他并行 migration 冲突。
3. `docs/fixtures/imports/` 已作为 parser testdata 来源。
4. OpenAPI 草案已被前后端共同确认。
5. `commit` 入口在 Task47 阶段不可见或关闭。
