## 4. API 行为

### 4.1 `POST /api/imports/preview`

请求：

```text
multipart/form-data
file: CSV
metadata: {"source_type":"wechat","mapping":{}}
```

成功：

```text
201 Created
status = ready
transactions count 不变
```

失败：

| 场景 | 错误 |
|---|---|
| source_type 缺失 | VALIDATION_ERROR |
| 文件空 | IMPORT_FILE_INVALID |
| 表头不支持 | IMPORT_FILE_UNSUPPORTED |
| 行数超限 | IMPORT_BATCH_TOO_LARGE |
| 非 owner | FORBIDDEN |

### 4.2 `GET /api/imports/{batch_id}`

返回 batch detail 和 row 列表。

要求：

1. 只允许当前账本 owner 访问。
2. 非成员不得泄露 batch 是否存在。
3. 已过期批次返回 `expired` 状态。

### 4.3 `PATCH /api/imports/{batch_id}/rows/{row_id}`

允许调整：

1. selected category/account/tags/visibility。
2. target_transaction_type。
3. row_status: pending/adjusted/skipped。

限制：

1. committed batch 不可修改。
2. 归档元数据不得静默应用。
3. 保存 row 不创建 transaction。

## 6. Task47U UI/UX 切片

Task47 的 UI/UX 不延后到 Task48/Task49 收尾。Task47U 与 Task47 后端并行推进，目标是先建立可用的导入预览工作台骨架。

必须完成：

1. Import Entry：来源选择、上传区、格式错误提示和 owner-only 状态。
2. Preview Workbench：批次统计、行状态汇总和行列表。
3. Import Row Card：移动端核心字段卡片，显示行号、金额、商户/标题、状态和错误摘要。
4. 状态系统：`new`、`duplicate`、`suspicious`、`invalid` 使用统一文案和非纯颜色识别。
5. Task47 阶段不展示可提交的 commit 主按钮；如保留位置，必须 disabled 且说明“预览阶段暂不可提交”。

不进入 Task47U：

1. 提交确认弹窗，归 Task48U。
2. 导入结果页，归 Task48U。
3. 规则管理页，归 Task49U。
4. 全局浅色主题迁移，仍作为 v1.2-v1.3+ 长期专项评估。

验收：

1. 375px 导入预览无横向滚动。
2. invalid 行不只依赖颜色表达错误。
3. 用户能明确知道 preview 未写入 `transactions`。
4. Figma 设计稿或设计规格同步到 `docs/ui/figma/`。
