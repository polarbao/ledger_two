## 2. Feature flag semantics

`IMPORT_CLASSIFICATION_MODE` 只接受以下值，未知值启动失败：

| Mode | Preview behavior | Commit behavior | Intended use |
|---|---|---|---|
| `off` | 保留 Task49 既有单规则建议，不运行 Task53 classifier | 使用已持久化 selected/suggested，不重分类 | 默认值、代码回退 |
| `suggest` | 运行 Task53 classifier 并持久化解释；所有 classifier 结果只写 suggested，fallback 也不自动写 selected | 不重分类 | staging 观察与误命中评审 |
| `graded` | high auto 写 selected；medium/low 写 suggested；fallback 写兜底 selected | 不重分类 | 通过验收后的候选行为 |

共同约束：manual/bulk 结果始终受保护；duplicate/invalid/skipped 不进入 classifier；切换开关不会反向改写已提交 transaction。

## 3. Task53.3 contract freeze

### 3.1 Persistence mapping

| Classifier decision | import_items writes |
|---|---|
| `auto_selected` | `selected_*`；兼容 `suggested_*`；status/confidence/source/reason/matched rules |
| `suggested` | 仅 `suggested_*`；status/confidence/source/reason/matched rules |
| `fallback` | graded 写 `selected_category_id`；suggest 只保留建议与解释 |
| `conflict` | 不写分类选择；持久化冲突原因和 matched rules |
| `manual` / `bulk` | 保留当前 selected，不由 reclassify 覆盖 |
| `unresolved` | 不伪造分类或命中规则 |

`classification` DTO 只承载状态、来源、置信度、解释、命中规则和 suggested 投影。最终值继续使用 PreviewRow 已有 `selected_category_id`、`selected_account_id`、`selected_tag_ids`；禁止在嵌套 DTO 再复制一套 selected 真值。

### 3.2 Repository transaction boundary

1. preview 在一个 batch 事务中读取一次 classification context，逐行计算并持久化，不为每行重复查询规则。
2. parser、`file_sha256`、`import_hash` 和 duplicate 判定顺序不变；分类在 eligible 判定之后执行。
3. commit 只读取 preview 持久化结果，规则在 preview 后变化不得影响同一批次提交。
4. reclassify dry-run 不写库；执行模式只更新 ready batch 的 eligible 且非 manual/bulk 行。
5. summary 由持久化行状态聚合，不使用前端传入计数。

### 3.3 Reclassify response

返回 `dry_run`、eligible/changed/unchanged、protected manual/bulk、conflict 和各 classification status 计数。执行前后的差异只包含行 ID、旧/新 status 与旧/新最终分类/标签；不返回真实账单原文作为审计 payload。

## 4. Task53.4 contract freeze

### 4.1 Bulk adjustment

1. `accept_suggestions` 逐行接受各自持久化建议；不接受客户端临时建议值。
2. `apply_values` 对选中行写统一分类、账户和标签；允许解决 conflict。
3. invalid/duplicate/skipped 永不修改；响应按 updated/skipped/conflict/error 行 ID 分类，支持 UI 定位。
4. 成功行标记 `bulk`；批量请求不创建 learned rule。
5. 事务以单次 bulk 请求为边界：参数或权限错误整次失败，行状态不可处理则进入 skipped/errors，不产生半条行更新。

### 4.2 Explicit learning

1. learn API 只接受已保存行的最终 selected 值，规范化 merchant 后生成 `merchant_equals`。
2. 默认限定当前账本和当前来源；`all_sources` 必须由 Owner 显式选择。
3. 同一规范化 merchant/source 的 learned rule 幂等更新，不覆盖 manual rule。
4. 行保存与 learn 是两个请求；learn 失败不回滚已完成的行保存。
5. 空 merchant、跨账本对象、归档元数据、分类类型错配和标签超过 8 个返回稳定错误码。

### 4.3 Metadata safeguards

规则引用数必须来自当前账本；stale rule 不再参与候选。归档 `expense_other` 或 `income_other` 时必须在同一事务指定同类型替代分类，并更新 profile/fallback 指向；历史 transaction 分类不改写。
