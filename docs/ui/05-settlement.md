# UI：结算中心

## 页面目标

让用户清楚看到当前谁应该转给谁多少钱，以及结算明细和历史结算记录。

## 核心区域

- 当前结算关系。
- 应结算金额。
- 双方实际支付金额。
- 双方实际承担金额。
- 双方净额。
- 已结算记录。

## 主要操作

- 生成结算记录。
- 查看结算历史。
- 查看影响结算的共同支出明细。

## 交互规则

- 点击生成结算记录必须二次确认。
- 确认后生成 settlement 记录。
- 不修改历史账单。
- 结算成功后刷新当前待结算金额。

## 空状态

双方已结清时展示：

```text
当前双方已结清
暂无需要结算的金额
```
