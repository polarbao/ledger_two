# UI：结算中心

## 页面目标

让用户清楚看到当前谁应该转给谁多少钱，以及结算明细和历史结算记录。

## 核心区域

- 结算统计范围：全部未结、仅本月。
- 当前结算关系。
- 应结算金额。
- 双方实际支付金额。
- 双方实际承担金额。
- 共同支出净额、已登记结算和最终未结。
- 影响结算的共同支出入口。
- 已结算记录。

## 主要操作

- 生成结算记录。
- 复制结算文案。
- 查看结算历史。
- 查看影响结算的共同支出明细。

## 交互规则

- “全部未结”调用不带月份的 balance API；“仅本月”使用当前选择月份，不能只切换文案而复用全部范围结果。
- `paid/share/raw_net/settlement/final_net` 直接展示服务端结果，前端不重新实现权威结算计算。
- 复制成功或失败均不改变结算状态、不新增流水；失败时展示可手动选择的完整文本。
- 点击生成结算记录必须二次确认。
- 确认框显示付款方、收款方和金额，并明确生成 settlement 记录及对应流水。
- 确认按钮使用“生成结算记录”，不使用含义不清的“确定”。
- 生成记录不修改历史共同支出。
- 结算成功后刷新当前待结算金额。
- 同步刷新结算历史、Dashboard、流水和分析缓存。

## 空状态

双方已结清时展示：

```text
当前双方已结清
暂无需要结算的金额
```
