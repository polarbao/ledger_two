# UI：统计分析

状态：UI-FL-09 已实现并验收

实现入口：`frontend/src/pages/AnalyticsPage.tsx`

验收证据：`docs/project_analysis/ui-fl-09-analytics-2026-07-15/`

## 页面目标

统计页用于展示消费结构、趋势、成员承担关系、标签排行与结算趋势。

## Tab 结构

- 趋势：近六个月支出/收入、本月个人/共同支出和当前未结。
- 分类：按金额降序展示分类、占比并钻取流水。
- 成员：实际支付、消费承担、垫付净额、已登记结算和最终未结。
- 标签：按金额降序展示标签场景、占比并钻取流水。

结算不再作为消费分析 Tab。登记结算属于独立记录和独立页面，只在成员维度用于解释最终未结。

## 数据与口径

- 所有金额来自既有 reports API，前端不重算后端权威账单、分摊或结算。
- 趋势通过既有 monthly-summary 接口读取六个账期，不新增前端伪聚合接口。
- 支出包含 expense 与 shared_expense；income 单独展示，不与支出相抵。
- settlement 不进入消费支出、分类或标签统计，只调整成员最终未结。
- 标签采用全额命中口径，多标签账单会分别计入每个标签，标签总和不能用于校验账期总支出。
- 统计基于当前用户可见账单，private 不会泄漏给另一成员。

## 交互

- 点击分类进入筛选后的流水页。
- 点击标签进入标签筛选。
- 点击趋势月份进入对应账期流水。
- 点击成员“查看支付流水”进入 `payer_user_id` 筛选。
- 当前月份继续由 AppShell 月份控件管理；自定义范围未有后端契约，本阶段不伪造。

## 设计原则

移动端不堆叠复杂图表，优先展示关键数字、可读柱状图、Top 列表和成员卡片。375/1440px 必须无横向滚动，图表的每个月份同时提供可读金额和键盘可达钻取动作。
