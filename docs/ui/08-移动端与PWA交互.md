# UI：移动端与 PWA 交互

## 1. 目标

让 LedgerTwo 在手机浏览器中接近原生 App 使用体验，并为 PWA 和后续跨端客户端预留交互规范。

## 2. 移动端信息架构

底部 Tab：

- 首页。
- 流水。
- 记账。
- 结算。
- 我的。

记账入口建议位于底部 Tab 中间，或使用右下角悬浮按钮。

## 3. 移动端页面规则

- 首页只展示关键数字和最近流水，不堆叠复杂图表。
- 流水页使用卡片列表。
- 筛选使用底部弹窗。
- 详情使用底部抽屉或新页面。
- 记账表单使用底部弹窗，复杂场景可进入独立页面。

## 4. PWA 行为

- 支持添加到手机桌面。
- 支持启动图标。
- 支持静态资源缓存。
- 离线时展示只读或离线提示。
- 离线草稿后续版本支持。

## 5. 触控交互

- 账单卡片点击进入详情。
- 长按进入多选，后期可选。
- 左滑显示编辑 / 删除，谨慎实现，删除需二次确认。

## 6. 验收标准

- iPhone Safari 和 Android Chrome 可正常访问。
- 主要页面在 375px 宽度下不横向滚动。
- 记账、筛选、结算操作在手机端可完成。
- PWA 可添加到桌面，刷新后保持登录态。
