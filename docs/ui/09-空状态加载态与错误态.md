# UI：空状态、加载态、错误态与骨架屏

## 1. 文档目标

定义 LedgerTwo 在加载、无数据、请求失败、权限不足、服务异常和离线场景下的统一 UI 表现。

目标：

- 避免页面空白。
- 告诉用户当前发生了什么。
- 提供明确下一步操作。
- 保证 Web、移动端、PWA 体验一致。
- 为 AI 生成前端组件提供明确规则。

## 2. 状态类型

| 状态 | 说明 |
|---|---|
| loading | 数据请求中 |
| skeleton | 首屏或列表加载骨架屏 |
| empty | 请求成功但没有数据 |
| error | 请求失败 |
| unauthorized | 未登录或登录过期 |
| forbidden | 无权限访问 |
| offline | 当前网络不可用 |
| degraded | NAS 服务异常或数据库异常 |
| success | 操作成功反馈 |

## 3. 加载态

### 3.1 App 启动加载

用于检查初始化状态、登录态、当前用户信息。

文案：

```text
正在加载 LedgerTwo...
```

要求：居中展示 Logo 或文字 + 小型 Spinner，不允许出现空白页。

### 3.2 页面级加载

页面框架先展示，数据区域使用 skeleton，不要整页闪白。

Dashboard：5 个卡片 skeleton、最近流水 skeleton、分类 Top skeleton。

流水页：桌面端表格行 skeleton，移动端账单卡片 skeleton。

统计页：图表矩形 skeleton、Top 列表 skeleton。

设置页：分组列表 skeleton，危险操作按钮默认禁用。

## 4. 空状态

### 4.1 Dashboard 无账单

```text
本月还没有账单
点击「记一笔」，记录第一笔共同支出
```

主按钮：`记一笔`

### 4.2 流水列表为空

无筛选条件：

```text
还没有任何流水
你可以先记录一笔支出、收入或共同支出
```

有筛选条件：

```text
没有找到符合条件的流水
可以尝试清空筛选条件
```

按钮：`清空筛选`

### 4.3 结算中心已结清

```text
当前双方已结清
暂无需要结算的金额
```

辅助说明：

```text
新的共同支出产生后，这里会自动更新待结算金额
```

### 4.4 统计页无数据

```text
当前时间范围内没有可统计的数据
记账后将展示分类、成员和标签统计
```

### 4.5 备份列表为空

```text
暂未生成备份
建议在正式使用前先创建一次手动备份
```

按钮：`立即备份`

## 5. 错误态

### 5.1 网络请求失败

```text
加载失败，请检查网络后重试
```

按钮：`重试`

### 5.2 NAS 服务异常

```text
服务暂时不可用
请检查 NAS 上的 LedgerTwo 服务是否正常运行
```

辅助说明：

```text
可以尝试刷新页面，或进入 Docker 查看容器日志
```

### 5.3 数据库异常

```text
数据库暂时不可用
请检查 NAS 存储空间、数据库文件权限和服务日志
```

该状态属于高风险异常，不要只用 Toast，需要页面级提示。

### 5.4 未登录或登录过期

```text
登录状态已过期，请重新登录
```

行为：清理当前用户状态并跳转登录页。

### 5.5 无权限

```text
你没有权限查看或操作该内容
```

对于 private 账单，统一展示：

```text
账单不存在或你没有权限查看
```

### 5.6 表单提交失败

表单顶部展示：

```text
保存失败，请检查填写内容后重试
```

字段级错误展示在字段下方，例如：

```text
金额必须大于 0
```

## 6. 成功反馈

- 新增账单成功：`账单已保存`
- 删除账单成功：`账单已删除`
- 结算成功：`结算记录已生成`
- 备份成功：`备份已生成`

结算成功后必须刷新结算中心、结算历史和 Dashboard 待结算金额。

## 7. 离线状态

顶部或底部 Banner：

```text
当前处于离线状态，部分数据可能不是最新
```

v0.2-v0.6 初期策略：

- 不允许离线提交正式账单。
- 可以保留表单草稿，后续支持。
- 保存按钮禁用或提示联网后再试。

## 8. 组件建议

建议抽象：

```text
EmptyState
ErrorState
LoadingSpinner
SkeletonCard
SkeletonTable
PageState
NetworkStatusBanner
```

## 9. 验收标准

- 所有页面请求中都有 loading 或 skeleton。
- 所有空列表都有明确空状态。
- 所有失败请求都有重试入口。
- 未登录自动跳转登录页。
- 无权限不泄露 private 账单内容。
- NAS 服务异常有页面级提示。
- 移动端无横向溢出。
