# UI：数据安全与高风险操作确认

## 1. 文档目标

定义 LedgerTwo 中所有可能影响真实账务数据、安全性或结算结果的高风险操作确认交互。

目标：

- 防止误删账单。
- 防止误生成结算。
- 防止误导入、误恢复、误覆盖数据。
- 让用户明确知道操作影响范围。
- 保证所有高风险操作都有统一 UI 模式。

## 2. 高风险操作列表

| 操作 | 风险等级 | 是否需要确认 | 说明 |
|---|---:|---|---|
| 删除账单 | 高 | 是 | 会影响统计和结算 |
| 修改账单金额 | 中 | 条件需要 | 已结算或共同支出建议提示 |
| 修改共同支出分摊 | 高 | 是 | 会改变结算结果 |
| 生成结算记录 | 高 | 是 | 会抵扣当前净额 |
| 删除结算记录 | 高 | 是 | 会恢复待结算金额，后续功能 |
| 导出数据 | 中 | 是 | 涉及隐私数据 |
| 手动备份 | 中 | 是 | 生成敏感备份文件 |
| 恢复备份 | 极高 | 是，且需输入确认文本 | 会覆盖当前数据 |
| 批量导入 | 高 | 是 | 可能生成大量账单 |
| 批量删除 | 极高 | 是，且需输入确认文本 | 可能影响大量历史数据 |
| 退出登录 | 低 | 可选 | 普通确认即可 |

## 3. 通用确认弹窗结构

高风险弹窗必须包含：

```text
标题
风险说明
影响范围
操作后果
取消按钮
确认按钮
```

危险操作确认按钮使用 danger 样式。

按钮文案不能只写“确定”，必须写清动作，例如：删除账单、确认结算、开始导入、下载备份。

## 4. 删除账单确认

### 4.1 普通账单删除

标题：

```text
确认删除这笔账单？
```

说明：

```text
删除后，这笔账单将不再出现在流水和统计中。
```

按钮：取消 / 删除账单。

### 4.2 共同支出删除

标题：

```text
确认删除这笔共同支出？
```

说明：

```text
删除后，本月双方待结算金额会重新计算。历史结算记录不会被自动删除。
```

影响展示：金额、付款人、分摊方式。

按钮：取消 / 删除共同支出。

## 5. 修改金额 / 分摊确认

当修改共同支出金额、付款人、参与人或分摊方式时，应提示：

```text
本次修改会影响结算金额
保存后，系统会根据新的金额和分摊方式重新计算当前待结算金额。
```

按钮：继续编辑 / 保存修改。

如果该账单发生在已有结算之前，后续版本需要提示：

```text
这笔账单发生在已有结算之前，修改后可能导致历史结算与当前计算结果不一致。
```

## 6. 生成结算确认

标题：

```text
确认已完成转账？
```

说明：

```text
系统将生成一条结算记录，用于抵扣当前待结算金额。历史账单不会被修改。
```

信息展示：付款人、收款人、结算金额。

按钮：取消 / 确认结算。

成功后：关闭弹窗、刷新结算中心、刷新结算历史、刷新 Dashboard 待结算金额、展示 Toast：结算记录已生成。

## 7. 导出数据确认

标题：

```text
导出账务数据？
```

说明：

```text
导出的文件可能包含账单金额、备注、分类、标签和成员信息，请妥善保存。
```

选项：CSV 导出、JSON 全量导出、SQLite 备份下载。

按钮：取消 / 开始导出。

## 8. 手动备份确认

标题：

```text
立即创建手动备份？
```

说明：

```text
系统将基于当前数据库创建一个备份文件。备份文件可能包含完整账务数据，请确保 NAS 存储空间充足。
```

按钮：取消 / 立即备份。

成功后显示备份文件名和时间。

## 9. 恢复备份确认

恢复备份属于极高风险操作，v0.2 不建议直接做 UI 恢复。后续实现时必须使用三段式确认。

### 9.1 选择备份

展示：备份文件名、备份时间、文件大小、账单数量、用户数量。

### 9.2 风险提示

```text
恢复备份会覆盖当前数据
恢复后，当前数据库中的账单、结算、分类、标签和设置将被备份内容替换。此操作不可直接撤销。
```

### 9.3 输入确认文本

要求用户输入：

```text
确认恢复
```

确认按钮在输入完全匹配前禁用。恢复前系统必须自动创建当前数据备份。

## 10. 批量导入确认

标题：

```text
确认导入这些账单？
```

说明：

```text
系统将导入预览列表中的账单。重复账单会被跳过，导入后可以在流水中查看。
```

展示：待导入条数、重复跳过条数、需要人工确认条数。

按钮：返回预览 / 确认导入。

## 11. 批量删除确认

批量删除属于极高风险操作。

标题：

```text
确认批量删除账单？
```

说明：

```text
这些账单将不再进入流水、统计和结算计算。此操作会影响历史数据。
```

要求输入：

```text
确认删除
```

按钮：取消 / 批量删除。

## 12. Toast 与 Modal 的使用边界

只用 Toast 的场景：保存成功、复制成功、筛选已清空、备份已生成。

必须使用 Modal 的场景：删除、结算、导出、备份、恢复、批量导入、批量删除。

## 13. 审计日志提示

对用户可展示轻量提示：

```text
此操作会记录到审计日志中
```

适用：删除账单、修改共同支出、生成结算、导出数据、手动备份。

## 14. 移动端交互

移动端高风险确认建议使用底部 Sheet 或全屏 Modal。

要求：

- 危险按钮不应贴近取消按钮，避免误触。
- 删除按钮使用红色。
- 恢复备份、批量删除必须要求输入确认文本。
- 弹窗内容过长时允许滚动，但确认按钮固定在底部。

## 15. 验收标准

- 删除账单必须有二次确认。
- 删除共同支出明确提示会影响结算。
- 生成结算明确显示付款人、收款人、金额。
- 导出和备份提示隐私风险。
- 恢复备份和批量删除需要输入确认文本。
- 所有危险按钮使用 danger 样式。
- 操作成功后刷新相关页面数据。
