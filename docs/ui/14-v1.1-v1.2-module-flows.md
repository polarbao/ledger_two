# UI：v1.1-v1.2 模块流程细化

状态：建议采纳  
适用阶段：Foundation before v1.1 之后

## 1. 目标

本文细化 v1.1 和 v1.2 的关键页面流程，确保产品模块落到可实现的界面和交互状态。

## 2. 快捷记账流程

入口：

- Dashboard 主按钮。
- 底部导航中央记账按钮。
- 流水页空状态按钮。

默认展示字段：

1. 金额。
2. 类型。
3. 分类。
4. 账户。
5. 时间。
6. 保存按钮。

折叠字段：

- 标题。
- 标签。
- 付款人。
- 参与人。
- 分摊方式。
- 可见性。
- 备注。

保存成功状态：

- Toast 显示“已保存”。
- 普通保存关闭表单。
- 保存并继续记保留分类、账户、付款人、可见性，清空金额和备注。

## 3. 复制与模板流程

账单详情操作：

- 复制一笔。
- 从此账单创建模板。

模板页入口：

- 设置 -> 模板与周期账单 -> 模板。
- 记账表单 -> 选择模板。

模板生成账单流程：

```text
选择模板
-> 打开记账表单并填入模板字段
-> 用户确认金额、时间和分类
-> 保存为真实账单
```

错误状态：

- 模板引用的分类已归档：提示替换或继续使用。
- 模板引用的账户已归档：要求替换。
- 当前用户无权限：禁用生成按钮并说明原因。

## 4. 周期账单流程

入口：

- Dashboard 待确认区域。
- 设置 -> 模板与周期账单 -> 周期账单。

待确认卡片展示：

- 名称。
- 金额。
- 到期日。
- 分类。
- 账户。
- 确认。
- 跳过。
- 编辑规则。

确认流程：

```text
点击确认
-> 打开账单确认表单
-> 用户可调整金额、时间、分类、备注
-> 保存后生成真实账单
```

跳过流程：

- 二次确认。
- 明确提示“只跳过本期，不删除规则”。

## 5. 分类、标签、账户管理流程

设置页结构：

```text
设置
├─ 账号与登录
├─ 账本与成员
├─ 分类
├─ 标签
├─ 支付账户
├─ 模板与周期账单
├─ 导入导出
├─ 备份恢复
└─ 系统诊断
```

列表操作：

- 新增。
- 编辑。
- 拖动排序或上下移动。
- 归档。
- 恢复。

归档确认：

- 若未被使用：提示归档后不再出现在新建表单。
- 若已被使用：提示历史账单仍会显示该名称。

## 6. 结算解释流程

结算页分区：

1. 当前结论：谁应转给谁多少钱。
2. 解释表：paid/share/raw_net/settlement/final_net。
3. 影响明细：共同支出和结算记录。
4. 操作区：生成结算记录、复制结算文案。

复制文案后：

- Toast 显示“已复制”。
- 不改变结算状态。
- 不新增通知记录。

## 7. 导入流程

入口：

- 设置 -> 导入导出 -> 导入账单。

步骤：

1. 选择来源：微信、支付宝、通用模板。
2. 上传文件：微信支持 CSV、XLSX，支付宝和通用模板仅 CSV。
3. 字段映射。
4. 预览列表。
5. 处理错误行和疑似重复行。
6. 确认导入。
7. 导入结果。

预览列表列：

- 行号。
- 时间。
- 商户/标题。
- 金额。
- 收支方向。
- 推荐分类。
- 推荐标签。
- 推荐账户。
- 状态。
- 操作。

XLSX 批次摘要额外展示：

- 文件格式。
- 识别工作表。
- 表头物理行号。
- 实际识别数据行数。

上传区不得只写“CSV 导入”；统一使用“账单文件导入”。格式不匹配、加密、超限、缺失表头和多候选工作表必须有独立可行动文案。

状态颜色：

- new：普通。
- duplicate：弱化。
- suspicious：警示。
- invalid：错误。

## 8. 移动端约束

- 375px 宽度不得横向滚动。
- 表格在移动端改为卡片列表。
- 筛选器使用 bottom sheet。
- 导入预览在移动端允许只显示核心字段，详情进入行详情页。
- 危险按钮和主按钮至少分隔一个视觉区块。
