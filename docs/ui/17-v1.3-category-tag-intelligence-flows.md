# UI：v1.3 分类、标签与导入智能归类流程

状态：Task53P UI/UX 基线；视觉审阅稿待 Task53U 前补齐<br>
创建日期：2026-07-16<br>
主题：Fresh Light 默认，Dark Glass 保留回退<br>
适用视口：375、390、430、1440

## 1. Goal

让用户在大批量导入时一眼区分“系统已经确定”“需要我确认”“只是兜底”“我已修改”，并能用少量批量操作完成分类和标签整理。界面必须强调结果可解释和批次仍未提交，不能把自动分类包装成不可质疑的 AI 结论。

## 2. Information architecture

```text
导入账单工作台
├─ 上传与解析摘要
├─ 分类完成度摘要
├─ 状态筛选
├─ 预览行/卡片
│  ├─ 最终分类与标签
│  ├─ 自动化状态
│  ├─ 命中原因
│  └─ 编辑/接受/同商户应用/跳过
├─ 批量分类操作栏
├─ 导入规则管理
└─ 确认提交

设置
├─ 分类管理
├─ 标签管理
├─ 导入规则
└─ 基础分类与标签包
```

## 3. Classification summary

现有总行数、新增、重复、疑似、错误摘要继续保留。新增第二行“整理状态”，避免业务状态与分类状态混为一组：

| 状态 | 文案 | 色彩语义 | 点击行为 |
|---|---|---|---|
| auto_selected | 已自动选择 | success | 筛选高置信自动结果 |
| suggested | 待接受建议 | info | 筛选可批量接受项 |
| fallback | 使用其他分类 | warning | 筛选需要重点复核项 |
| manual | 已手工调整 | neutral | 筛选用户确认项 |
| bulk | 已批量调整 | neutral | 筛选本批次批量确认项 |
| conflict | 分类冲突 | danger | 筛选必须处理项 |

数字变化通过 `aria-live=polite` 播报，不以颜色作为唯一状态表达。

## 4. Preview row

桌面表格核心列：

```text
行号 | 时间 | 商户/标题 | 金额 | 最终分类 | 标签 | 归类状态 | 操作
```

移动端卡片顺序：

1. 商户/标题与金额。
2. 日期和来源行号。
3. 分类和标签。
4. 状态 chip 与一行原因。
5. 操作按钮。

状态展示：

```text
[已自动选择] 餐饮 · 外卖
精确匹配已记住的商户「美团外卖」
```

```text
[待接受建议] 交通 · 通勤
商户名称包含「滴滴」
```

```text
[使用兜底] 其他支出
未找到可靠分类，请重点复核
```

禁止事项：

1. 不显示“AI 已完成”或虚假准确率。
2. 不只显示数据库 ID。
3. 不把原因藏在 hover-only tooltip；移动端和键盘用户必须可访问。
4. 不在预览卡片内再嵌套分类卡片。

## 5. Row editor

字段顺序：

1. 目标类型。
2. 分类，必选；按支出/收入过滤。
3. 标签，多选，显示 `已选 N/8`。
4. 账户。
5. 可见性。
6. `记住此商户的分类和标签` checkbox。
7. 保存。

记住商户控件只在以下条件显示：

1. 商户规范化后非空。
2. 行不是 duplicate/invalid/skipped。
3. 当前用户为 Owner。

勾选后显示范围摘要：

```text
以后在当前账本导入“滴滴出行”时自动选择这些内容。
```

若存在冲突规则，保存按钮仍可保存本行，但学习动作单独失败并提示“本行已保存，规则未创建”，避免把两个结果混成一次不可恢复失败。

焦点规则：关闭抽屉后回到原行“编辑”按钮；错误时焦点移到首个错误字段。

## 6. Bulk actions

### 6.1 Accept suggestions

入口：`待接受建议`筛选页顶部固定操作栏。

```text
已选择 18 条       [接受分类与标签建议]
```

确认弹窗展示：

1. 影响行数。
2. 将更新分类的行数。
3. 将新增标签的行数。
4. 跳过冲突/无效/重复数量。

不提供“一键接受全部后立即提交”。接受建议和提交导入必须是两个独立动作。

“接受建议”逐行采用各自已持久化的建议；同一批选择可以包含不同分类。需要统一设置相同分类/标签时，使用独立“应用相同值”动作。

### 6.2 Apply to same merchant

行操作：`应用到相同商户`。

Bottom sheet/dialog 展示规范化商户和命中行数，允许选择：

1. 仅本批次应用。
2. 本批次应用并记住以后导入。

默认选中“仅本批次应用”，防止无意创建长期规则。

## 7. Rule manager

现有规则管理增加：

1. 来源筛选：用户规则、学习规则。
2. 行为：自动选择、仅建议。
3. 匹配：商户精确、商户包含、描述包含、来源账户、金额区间。
4. 最近命中次数和最后命中时间，作为只读辅助信息。
5. 引用归档分类/标签时显示阻断 warning。

学习规则默认折叠在“系统为你记住的规则”分组，但仍可编辑和归档。不要隐藏规则或使用不可撤销的“清除学习记录”。

## 8. Default metadata profile

### 8.1 First setup

初始化页不增加额外步骤；`basic_cn_v1` 随初始化创建。成功页可显示分类/标签数量，但不要求逐项确认。

### 8.2 New ledger

创建账本表单增加 segmented control：

```text
基础分类与标签 | 空白账本
```

默认选择基础包。旁边提供信息按钮，打开非阻断说明 sheet，列出数量和可在设置中修改，不在页面堆砌长说明。

### 8.3 Existing ledger

设置 -> 分类/标签管理提供命令按钮：`补充基础分类与标签`。

流程：

```text
打开预览
-> 显示将创建/复用/冲突/跳过
-> 解决冲突
-> 确认应用
-> 成功摘要
```

冲突项必须逐项选择复用或跳过；不自动给用户对象绑定 system_key。

## 9. Metadata management

分类列表：

1. 支出/收入 tabs。
2. 图标、名称、使用次数、规则引用数、系统基础标识。
3. 排序、编辑、归档/恢复。
4. 兜底分类使用盾牌图标和 tooltip“用于无法识别的账单”。

标签列表：

1. 颜色 swatch、名称、使用次数、规则引用数。
2. 默认标签和用户标签视觉同层级，避免默认包喧宾夺主。
3. 标签数量较多时提供搜索，不首期引入标签分组 UI。

归档兜底分类时打开 replacement dialog，使用分类选择器，而不是危险确认弹窗直接继续。

## 10. Empty/error/loading states

| 状态 | 主文案 | 操作 |
|---|---|---|
| 无任何分类 | 还没有可用于导入的分类 | 应用基础包 / 创建分类 |
| 无标签 | 标签是可选项 | 创建标签 |
| 无建议 | 当前账单没有可靠建议 | 手工选择 / 保留其他分类 |
| 规则冲突 | 多条规则给出了不同分类 | 查看冲突规则 |
| 规则失效 | 规则引用的分类或标签不可用 | 修复规则 |
| 批量部分成功 | 已更新可处理行，冲突行保持不变 | 查看未处理行 |
| 学习失败 | 本行已保存，长期规则未创建 | 重试创建规则 |

加载使用稳定高度 skeleton，不让筛选条和提交栏跳动。

## 11. Accessibility

1. 触控目标至少 44x44。
2. 状态 chip 具有可读文本和 `aria-label`。
3. 分类/标签选择器支持键盘搜索、方向键、Enter 和 Escape。
4. 多选标签移除按钮必须朗读“移除标签 X”。
5. 批量操作后播报成功、跳过、冲突数量。
6. Fresh Light 和 Dark Glass 均满足 WCAG AA 文本对比度。
7. 375px 下无横向滚动，长商户和标签 `overflow-wrap:anywhere`。

## 13. Shared component ownership

| 组件 | 所有权 | 规则 |
|---|---|---|
| `StatusChip` | UI foundation | 只新增 tone/label，不加入业务判断 |
| `SegmentedControl` | UI foundation | 复用，不改布局契约 |
| `ConfirmDialog` | UI foundation | 仅用于真正确认；分类选择使用专用 dialog |
| `ImportRowEditor` | Task53 | 增加分类状态和 remember control |
| `ImportPreviewRows` | Task53 | 增加解释和批量选择 |
| `MetadataManagePage` | Task53 | 增加系统键/规则引用/基础包入口 |
| `LedgerCreate` | Task50 外壳已冻结；Task53U 负责 profile 增量 | 复用现有状态机与管理 UI，只增加 profile 选择/预览，不重写 Task50 生命周期和成员流程 |

## 14. Acceptance

1. 用户可以按自动、建议、兜底、手工、冲突筛选。
2. 每行自动结果有非 hover 的可读解释。
3. 单条编辑、接受建议、同商户批量应用和记住商户均可撤销或修改。
4. 375/390/430/1440 下操作栏、标签和长商户不重叠。
5. 接受建议不会自动提交批次。
6. 规则学习失败与本行保存结果分开反馈。
7. 新建账本默认基础包，空白账本需显式选择。
8. 兜底分类归档必须选择替代分类。
