# UI 设计与开发交接清单

状态：执行要求，持续维护<br>
适用范围：每次 UI 设计稿、Figma 文件、前端实现或 UX 专项提交

## 1. 设计稿完成条件

| 检查项 | 要求 |
|---|---|
| 来源 | 明确 PRD、UI 文档或蓝本来源 |
| Frame | 标注页面名、版本、视口尺寸 |
| 状态 | 覆盖默认、加载、空、错误、权限不足 |
| 移动端 | 至少 375px 或 390px 设计稿 |
| 桌面端 | 工作台页面至少 1440px 设计稿 |
| 组件 | 能映射到组件库或明确需要新增 |
| Token | 使用本目录 token，不随意新增颜色和圆角 |
| 文案 | 高风险操作文案明确，不写模糊“确定” |

## 2. 开发交接条件

1. 已列出前端文件或建议新增组件位置。
2. 已说明是否影响 API、权限、金额、导入或备份。
3. 已说明是否需要迁移数据或新增后端字段。
4. 已说明移动端断点和无横向滚动要求。
5. 已说明验收命令或手工验收路径。

## 3. v1.1 收口交接

v1.1 只接受以下 UI/UX 改动：

1. 修复阻断真实使用的问题。
2. 修复移动端布局溢出、文字遮挡、金额挤压。
3. 修复状态不可解释、权限不可见、高风险操作不清晰。
4. 补齐验收证据。

不接受：

1. 全局主题迁移。
2. AppShell 大改。
3. 大面积组件重构。
4. 未经验证的新交互动效。

## 5. 验收证据

每次提交至少保留：

1. 文档或设计稿文件路径。
2. 变更摘要。
3. `git diff --check` 结果。
4. 若改代码，补前后端测试命令。
5. 若改 UI，补 375px 或 390px 截图/CDP 指标。
