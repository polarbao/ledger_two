## 3. Import Rule Manager

结构：

1. 顶部标题：导入规则、启用数量。
2. 状态筛选：全部、启用、归档。
3. 表单：规则名称、匹配类型、匹配内容、推荐分类、推荐账户、推荐标签、优先级、保存按钮。
4. 规则列表：名称、匹配条件、优先级、推荐结果、编辑、归档或恢复。
5. 空状态：未创建规则时显示轻提示。

交互：

1. 创建和编辑共用表单。
2. 编辑规则时必须回填分类、账户、多标签和优先级。
3. 标签使用多选 chip/checkbox，不再使用单选下拉。
4. 归档规则保留在列表中，但视觉弱化。
5. 归档规则不参与预览命中。

移动端约束：

1. 390px 下表单单列。
2. 标签 chip 可换行，不出现横向滚动。
3. 规则卡片操作按钮可换行或双列排列。
4. 长规则名、长标签名必须 `anywhere` 换行。

## 4. Rule Hit Explanation

预览行命中规则时展示：

1. 后端解释文案，例如“命中规则：星巴克”。
2. 推荐分类名称。
3. 推荐账户名称。
4. 推荐标签名称，可多标签。

不展示：

1. 不展示数据库 ID 作为主文案。
2. 不暗示规则已经自动写入正式账单。
3. 不替用户自动确认疑似重复行。

优先级：

1. 有 `suggestion_reason` 时作为第一行。
2. 有推荐元数据时作为第二行，格式为 `建议：分类 餐饮 · 账户 微信 · 标签 咖啡、报销`。
3. 元数据名称缺失时允许降级展示 ID 前 8 位，但应在后续归档元数据提示中优化。

## 5. Rule Metadata Warning

目标：规则引用归档分类、账户或标签时，不做静默推荐。

推荐设计：

1. 规则列表中显示 warning chip：`引用已归档元数据`。
2. 编辑表单中保留原值，并提示用户替换为启用元数据。
3. 预览行不自动应用归档元数据；若后端仍返回，应显示“需替换推荐项”。

本轮开发状态：

1. 当前前端查询 active 元数据用于新建和编辑推荐项。
2. 规则列表已显示归档或不可用元数据提示。
3. Figma 已绘制 `Rule Metadata Warning / Fresh Light`。
