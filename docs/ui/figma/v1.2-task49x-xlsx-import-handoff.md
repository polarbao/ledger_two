## 1. 设计目标

Task49X 不新建独立 Excel 页面，而是在现有导入工作台补齐“格式选择、解析事实、结构错误”三类状态。设计必须让用户理解 XLSX 只是输入容器，预览、去重、调整和提交仍是同一条安全链路。

## 3. 上传组件变体

`ImportFileDropzone` 增加：

```text
source=wechat|alipay|generic
state=idle|dragover|selected|uploading|parsing|error|disabled
format=csv|xlsx|unknown
```

内容规则：

1. 主动作：“选择账单文件”。
2. 微信/支付宝辅助文案：“CSV 或 XLSX，最大 2 MB，单批最多 2000 行”。
3. 通用模板辅助文案：“仅支持 LedgerTwo 通用 CSV”。
4. `selected` 展示文件名、格式图标和文件大小，不显示本地绝对路径。
5. `parsing` 使用稳定高度进度状态，不承诺百分比。
6. `IMPORT_XLSX_ENABLED=false` 时，微信/支付宝只接受 CSV，辅助文案显示“当前环境仅开放 CSV”，不展示不可执行的 XLSX 承诺。

## 4. 解析摘要组件

新增 `ImportParserSummary`，使用紧凑 definition list，不使用嵌套卡片：

```text
文件格式   XLSX
工作表     Sheet1
表头位置   第 18 行
识别流水   276 条
```

CSV 只显示文件格式和识别流水；不展示空工作表字段。摘要位于批次统计之前，不能抢占主状态计数的视觉层级。

## 5. 错误状态

错误面板统一包含：

1. 明确标题，不只显示错误码。
2. 一句原因。
3. 一条用户可执行的修复建议。
4. “重新选择文件”主动作。
5. 可折叠技术详情，只显示错误码、候选工作表或缺失字段，不显示原始账单内容。

颜色不是唯一识别手段；错误图标、标题和文案必须同时存在。

## 6. 响应式约束

1. 375px/390px 无横向滚动。
2. 文件名使用中间省略并保留扩展名。
3. 解析摘要在移动端使用两列 definition list，不压缩为横向标签组。
4. 候选工作表名称过长时换行，不扩大弹窗宽度。
5. 主按钮固定为单行；中文最长文案仍需完整显示。
